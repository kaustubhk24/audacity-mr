नायक्विस्ट

ओड्यासिटीआपला स्वतःचा प्लग-इन प्रभाव लिहिण्यासाठी नायक्विस्ट प्रोग्रामिंग भाषा वापरण्याची परवानगी देतो. व्हीएसटी आणि एलएडीएसपीए प्लगइनच्या विपरीत, न्यक्विस्ट प्लगइन सामान्य मजकूर संपादकाचा वापर करुन लिहिले जाऊ शकतात आणि ते संकलित करण्याची आवश्यकता नाही.

नायक्विस्ट लिस्प मांडणी आणि एसएएल नावाच्या अधिक पारंपारिक वाक्यरचना दोन्हीचे समर्थन करते. 1.3.8 पूर्वीची धृष्टता आवृत्त्या केवळ लिस्पला समर्थन देतात, परंतु सद्य ओड्यासिटी2.0 मालिका लिस्प आणि एसएएल दोन्हीचे समर्थन करते. ओड्यासिटीसह वापरासाठी प्लग-इन्स लिहिण्यासाठी आपल्या ओड्यासिटीच्या आवृत्तीसाठी प्राधान्य आणि योग्य मांडणीसाठी योग्य न्यक्विस्ट मार्गदर्शक-पुस्तिका निवडा:

  • नायक्विस्ट आवृत्ती 2.37 महितीपुस्तिका - संपूर्णपणे लिस्प वाक्यरचना वापरुन
  • नायक्विस्ट 3.1x संदर्भ महितीपुस्तिका - नवीनतम नायक्विस्ट मार्गदर्शक-पुस्तिका (मुख्यतः एसएएल वाक्यरचना वापरुन).

लक्षात ठेवा ओड्यासिटीसह वापरण्यासाठी आपल्याला साधे प्लग-इन लिहिण्यासाठी नायक्विस्ट डाउनलोड करण्याची आवश्यकता नाही. आपल्याला लिस्प आणि २.3737 महितीपुस्तिकासह कार्य करण्यासाठी आवश्यक असलेल्या सर्व मूलभूत सूचना खाली आहेत. आपण एसएएल देखील एक्सप्लोर करू इच्छित असल्यास आणि ओड्यासिटीमधील नवीनतम न्युक्विस्ट वैशिष्ट्यांसाठी , विकीवर नायक्विस्ट डॉक्युमेंटेशन पहा .

( प्रथम कालावधी ) प्रथम ( कालावधी ) यादीचा पहिला घटक (कार)
( बाकीची मुदतवाढ ) विश्रांती ( संप ) उर्वरित यादी (सीडीआर)
( रिव्हर्स एक्सप्रेस ) उलट ( कालावधी ) यादी उलट करा
( यादी एक्सप्रेस ...) यादी ( एक्सप्रेस ...) अभिव्यक्तींची यादी तयार करा
( जोडा एक्सप्रेस ...) परिशिष्ट ( एक्सप्रेस ...) दोन किंवा अधिक याद्या जोडा
( लांबी वाढवणे ) लांबी ( एक्सप्रेस ) यादीची लांबी
( नकाशाची यादी एफसीएन यादी 1यादी ...) नकाशाची यादी ( एफसीएन यादी 1 यादी ...) यादीतील प्रत्येक घटकाला कार्ये लागू करा


सशर्त अभिव्यक्ती

लिस्प शेल वर्णन
( जर टेस्ट-एक्सपायर ट्रू-एसएमटी [ खोटे-एसएमटी ]) जर चाचणी- एक्सप्रेशन केले तर सत्य-एसएमटी [ अन्यथा खोटे-एसएमटी ] जर टेस्ट-एक्सपायर खरे असेल तर ट्रू-एसएमटीचे मूल्यांकन करेल, अन्यथा चुकीचे-एसएमटीचे मूल्यांकन करेल
( जेव्हा टेस्ट-एक्सपायर विधान ) जेव्हा टेस्ट-एक्सपायर विधान जेव्हा टेस्ट-एक्सपायर खरे असेल तर विधानचे मूल्यांकन करा
( तोपर्यंत चाचणी-एक्सप्रेस विधान ) तोपर्यंत चाचणी-एक्सप्रेस विधान जेव्हा टेस्ट-एक्सपायर चुकीचे असते तेव्हा विधानचे मूल्यांकन करा
( स्थिती [( प्रेड1 एक्सप्रेस1 ) [( प्रेड2 एक्सप्रेस2 ) ...]]) जेव्हा प्रेड 1 सत्य असेल, तर एक्सप्रेस 1 चे मूल्यांकन करा, अन्यथा जर प्रेड 2 खरे असेल ...
( केस टेस्ट-एक्सप्रेस [( मूल्य विधान ) ...]) चाचणी-एक्सप्रेसशी जुळणार्‍या पहिल्या मूल्यासाठी विधानाचे मूल्यांकन करा
