४ ४ The proprietary ASIO interface standard is essential on Windows for low latency recording and playback. It is also usually the best way of making Multi-channel recordings on Windows. विंडोजवर कमी विलंब मुद्रण आणि प्लेबॅकसाठी मालकीचे एएसआयओ संवादपटल मानक आवश्यक आहे. विंडोजवर बहु-वाहिनी मुद्रण बनविण्याचा हा देखील एक चांगला मार्ग आहे.
५ ५ Licensing restrictions prevent us including ASIO support in released versions of Audacity, but Audacity can be compiled with ASIO support for private, non-distributable use. परवाना प्रतिबंध आम्हाला ओड्यासिटीच्या प्रकाशन झालेल्या आवृत्त्यांमध्ये एएसआयओ समर्थनासहित प्रतिबंधित करते, परंतु खाजगी, वितरित न करण्यायोग्य वापरासाठी एएसआयओ समर्थनासह ओड्यासिटी संकलित केली जाऊ शकते.
६ ६ This page summarizes ASIO licensing issues and steps to compile Audacity with ASIO support. हे पृष्ठ एएसआयओ परवाना देणार्‍या समस्यांचा आणि एएसआयओ समर्थनासह ओड्यासिटीचे संकलन करण्याच्या चरणांचे सारांश देते.

११ ११ Latencies on Windows, Linux and Mac विंडोज, लिनक्स आणि मॅकवरील उशिरा
१२ १२ ASIO is a proprietary audio interface standard in use on Windows which bypasses the operating system's mixing kernel, so providing lowest latency direct communication between computer audio software and hardware. एएसआयओ हे विंडोजवरील वापरात असलेले मालकीचे ध्वनी संवादपटल मानक आहे जे कार्यप्रणालीच्या मिश्र कर्नलला मागे टाकत आहे, म्हणून संगणक ध्वनी अनुप्रयोग आणि हार्डवेअर दरम्यान सर्वात कमी विलंब थेट संवाद प्रदान करते.
१३ १३ ASIO supports २४-bit sampling which is only otherwise available under Windows WASAPI or WDM-KS (Windows Driver Model Kernel Streaming). २४-bit sampling allows greater dynamic range, lower theoretical noise floor and greater resolution at lower audible volumes. एएसआयओ २४-बिट सॅम्पलिंगचे समर्थन करतो जे फक्त अन्यथा विंडोज डब्ल्यूएएसएपीआय किंवा डब्ल्यूडीएम-केएस (विंडोज ड्राइव्हर मॉडेल कर्नल स्ट्रीमिंग) अंतर्गत उपलब्ध आहे. २४-बिट सॅम्पलिंग मोठ्या गतिमान श्रेणी, कमी सैद्धांतिक ध्वनी मजला आणि कमी ऐकण्यायोग्य खंडांवर अधिक रिझोल्यूशनची परवानगी देते.
१४ १४ An unmixed ASIO output is "bit identical" to the original source. मूळ नसलेल्या एएसआयओ उत्पादित मूळ स्त्रोतापेक्षा "बिट एकसारखे" आहे.
१५ १५ Multiple physical input and output channels of the hardware are accessed over one single device. हार्डवेअरचे एकाधिक भौतिक माहिती पुरवणे आणि उत्पादित वाहिनी एकाच उपकरणावर प्रवेश केले जातात.
१६ १६ Audacity includes support for Microsoft's Windows DirectSound interface protocol. To use it, select "Windows DirectSound" as host in Device Toolbar. This will give support for multi-channel recording on some sound devices, but not the very low latencies that are possible on ASIO. ओड्यासिटीमध्ये मायक्रोसॉफ्टच्या विंडोज डायरेक्टसाऊंड संवादपटल शिष्टाचारचे समर्थन समाविष्ट आहे. ते वापरण्यासाठी, उपकरणे साधनपट्टीमध्ये यजमान म्हणून "विंडोज डायरेक्टसाऊंड" निवडा. हे काही ध्वनी उपकरणांवर बहु-वाहिनी मुद्रणला समर्थन देईल, परंतु एएसआयओवर शक्य तितक्या कमी विलंब नाही.
१७ १७ On Linux, the standard ALSA audio API typically provides lower latencies than Windows under MME or Windows DirectSound. However, many Linux distributions now use PulseAudio by default for audio routing and mixing. PulseAudio sits between the sound source and the Linux kernel and thus has somewhat higher latency than direct use of ALSA. For lowest latencies, you can use the JACK API that provides both low latency audio communication and audio routing between applications. Current Audacity supports JACK fairly well, but with some limitations. लिनक्स वर, मानक ALSA ध्वनी API सामान्यत: MME किंवा Windows DirectSound अंतर्गत Windows पेक्षा कमी विलंब प्रदान करते. तथापि, बर्‍याच लिनक्स वितरणे आता ध्वनी रूटिंग आणि मिक्सिंगसाठी पूर्वनिर्धारितनुसार पल्स ध्वनी वापरतात. पल्स ध्वनी आवाज स्रोत आणि लिनक्स कर्नल दरम्यान बसला आहे आणि म्हणूनच ALSA च्या थेट वापरापेक्षा काही प्रमाणात उशीर झाला आहे. सर्वात कमी विलंबसाठी, आपण जॅक एपीआय वापरू शकता जे अनुप्रयोगांदरम्यान कमी विलंबता ध्वनी संप्रेषण आणि ध्वनी रूटिंग दोन्ही प्रदान करते. सध्या ओड्यासिटी JACK ला बऱ्यापैकी समर्थन देते, परंतु काही मर्यादांसह.
१८ १८ On Mac, Core Audio is the standard API and is fully supported by Audacity. Core Audio also has lower latencies than Windows under MME and Windows DirectSound but Jack OS X can be used for lowest latency. मॅक वर, मूळ ध्वनी एक मानक एपीआय आहे आणि ओड्यासिटीद्वारे पूर्णपणे समर्थित आहे. मूळ ध्वनीमध्ये एमएमई आणि विंडोज डायरेक्टसाऊंड अंतर्गत विंडोजपेक्षा कमी विलंब देखील आहेत परंतु सर्वात कमी विलंब होण्यासाठी जॅक ओएस एक्सचा वापर केला जाऊ शकतो.
१९ १९ Audacity and ASIO ओड्यासिटी आणि एएसआयओ
२० २० The ASIO technology was developed by German company Steinberg and is protected by a licensing agreement which prevents redistribution of its source code. एएसआयओ तंत्रज्ञान जर्मन कंपनी स्टेनबर्गने विकसित केले आहे आणि परवाना कराराद्वारे संरक्षित आहे जे त्याच्या स्त्रोत कोडचे पुनर्वितरण प्रतिबंधित करते.
२१ २१ Audacity, as an open source program licensed under the GPL, is therefore currently unable to support ASIO, despite being ASIO-capable (providing the user's sound device is similarly capable). If ASIO support were distributed in Audacity builds this would either violate Steinberg's licence agreement if the code were included, or conversely would violate Audacity's GPL Licence if the code were withheld. There are persistent rumours of Steinberg opening up licensing, but without any apparent movement. Anyone who cares about this issue is invited to make their views known to Steinberg via their Contact page. ओडीसिटी, जीपीएल अंतर्गत परवानाकृत मुक्त स्रोत प्रोग्राम म्हणून, सध्या एएसआयओ-सक्षम असूनही वापरकर्त्याचे समर्थन करण्यास असमर्थ आहे (वापरकर्त्याचे ध्वनी यंत्र प्रदान करणे देखील तितकेच सक्षम आहे). जर एएसआयओ समर्थन ओड्यासिटी नेतयार केलेले वितरित केला गेला असेल तर हे एकतर कोडचा समावेश केल्यास स्टीनबर्गच्या परवान्याच्या कराराचे उल्लंघन करेल किंवा कोड न ठेवल्यास उलटपक्षी ओड्यासिटीच्या जीपीएल परवान्याचे उल्लंघन करेल. स्टीनबर्ग कायम परवाना उघडण्याच्या अफवा पसरवत आहेत पण कोणतीही उघड हालचाल न करता. ज्या कोणालाही या समस्येची काळजी आहे त्यांना स्टेनबर्गला त्यांचे संपर्क पृष्ठाद्वारे त्यांची मते जाणून घेण्यासाठी आमंत्रित केले आहे.
२२ २२ Non-distributable ASIO support in Audacity ओड्यासिटीमध्ये न वितरणीय एएसआयओ समर्थन
२३ २३ Audacity provides ASIO support on Windows for individuals who are prepared to compile Audacity from source code using the optional Steinberg ASIO SDK. ओड्यासिटी विंडोजवर अशा व्यक्तींसाठी एएसआयओ समर्थन प्रदान करते जे पर्यायी स्टीनबर्ग एएसआयओ एसडीके वापरून स्त्रोत कोडमधून ओड्यासिटी संकलित करण्यास तयार आहेत.
२४ २४ ASIO support is provided strictly on the basis that it is NON-DISTRIBUTABLE, that is, you may NOT copy or distribute builds including ASIO support to anyone else. The build is strictly for your own personal (private or commercial) use. For the same reasons, Audacity can NOT distribute builds of Audacity including ASIO support, so please don't ask! ASIO समर्थन कठोरपणे प्रदान केले जाते की ते विना-वितरित आहे, म्हणजेच, आपण इतर कोणालाही ASIO समर्थनासह तयार केलेल्याची प्रत किंवा वितरण करू शकत नाही. तयार केलेले कठोरपणे आपल्या स्वत: च्या वैयक्तिक (खाजगी किंवा व्यावसायिक) वापरासाठी आहे. त्याच कारणांसाठी, ओड्यासिटी एएसआयओ समर्थनासह ऑड्यासिटीचे बिल्ड वितरित करू शकत नाही, म्हणून कृपया विचारू नका!
२५ २५ The following is an overview of compiling Audacity from source code including ASIO support. खाली ASIO समर्थनासह स्त्रोत कोडवरून ओड्यासिटी संकलित करण्याचे विहंगावलोकन आहे.
२६ २६ Install the free Microsoft Visual Studio Community Edition Integrated Development Environment (IDE). विनामूल्य मायक्रोसॉफ्ट व्हिज्युअल स्टुडिओ कम्युनिटी एडिशन एकात्मिक विकास पर्यावरण (आयडीई) स्थापित करा.
२७ २७ Download and install the ASIO SDK from Steinberg. स्टीनबर्ग वरून एएसआयओ एसडीके डाउनलोड आणि स्थापित करा.
२८ २८ Download the source code of the latest Audacity release from https://github.com/audacity/audacity/releases. Follow the steps in win/compile.txt in the source code to: Https://github.com / ओड्यासिटी / ओड्यासिटी / रीलीज पासून नवीनतम ओड्यासिटी रीलिझचा स्त्रोत कोड डाउनलोड करा. स्त्रोत कोड मधील win / compile.txt मधील चरणांचे अनुसरण कराः
२९ २९ Download and install the wxWidgets GUI toolkit then build wxWidgets using Visual Studio. डब्ल्यूएक्सविजेट्स जीयूआय साधनकिट डाउनलोड आणि स्थापित करा आणि नंतर व्हिज्युअल स्टुडिओचा वापर करुन डब्ल्यूएक्सविजेट्स तयार करा.
३० ३० Set the WXWIN environment variable to the directory where you installed Widgets and set the ASIOSDK_DIR environment variable to the directory where you installed the ASIO SDK. Reboot the computer. आपण ज्या ठिकाणी विजेट्स स्थापित केले त्या डिरेक्टरीमध्ये डब्ल्यूएक्सडब्ल्यूएन एनवायरनमेंट व्हेरिएबल स्थापित करा आणि जिथे आपण एएसआयओ एसडीके स्थापित केले त्या निर्देशिकेमध्ये एएसआयओएसडीके_डिआयर एन्व्हायर्नमेंट व्हेरिएबल स्थापित करा. संगणक रीबूट करा.
३१ ३१ Build Audacity using Visual Studio. व्हिज्युअल स्टुडिओ वापरुन ओड्यासिटी बिल्ड करा.
३२ ३२ Follow the instructions and download links on Developing On Windows in the Audacity Wiki for full details. संपूर्ण तपशीलांसाठी ओड्यासिटी विकीमध्ये विकसक ऑन विंडोज वरील सूचनांचे अनुसरण करा आणि दुवे डाउनलोड करा.
३३ ३३ If you need help, please ask on the Compiling Audacity board on the Audacity Forum. आपल्याला मदतीची आवश्यकता असल्यास, कृपया ओड्यासिटी मंचवरील कंपाईल ओड्यासिटी बोर्डवर विचारा.
