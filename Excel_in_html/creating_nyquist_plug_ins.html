11 11 Creating Nyquist Plug-ins Nyquist प्लगइन तयार करत आहे
12 12 Creating a plug-in for Audacity using Nyquist is as simple as creating a text file with the extension “.ny” with some Nyquist code, adding a few comments to indicate the type of plug-in, and placing the file in Audacity’s plug-ins directory. Here’s a very simple plug-in as an example: ऑडिसीटीचा वापर करून नेक्विस्टचा वापर करून प्लग-इन तयार करणे इतके सोपे आहे की काही नेयकिस्ट कोडसह “.ny” विस्तारासह मजकूर धारिका तयार करणे, प्लग-इनचा प्रकार सूचित करण्यासाठी काही टिप्पण्या जोडणे आणि धारिका प्लग- मध्ये धारिका ठेवणे. ins निर्देशिका. येथे उदाहरण म्हणून एक अगदी सोपी प्लग-इन आहे:
13 13 ;nyquist plug-in ; nyquist प्लग-इन
14 14 ;version 1 ; आवृत्ती 1
1५ 1५ ;type process टाइप प्रक्रिया
1६ 1६ ;name "Fade In" ; नाव "फेड इन"
1७ 1७ ;action "Fading In..." ; क्रिया "फेड इन इन ..."
1८ 1८ (mult (ramp) s) (अनेक (उतारा)
1९ 1९ The first line of a Nyquist plug-in must be exactly as in the example above, and the second line must indicate a version number. Version 2 and 3 plug-ins support more widgets, but version 3 plug-ins are not supported in Audacity 1.3.3 or earlier. The next line is the type of plug-in, which is discussed below. Then comes the name of the plug-in, which is what is displayed in the menu bar, and then the “action”, which is what Audacity displays while it is busy processing the plug-in. There are other optional lines that may follow. Any line that does not begin with a semicolon (;) is assumed to contain Nyquist code and will be executed. Nyquist प्लग-इनची पहिली ओळ वरील उदाहरणाप्रमाणेच असणे आवश्यक आहे आणि दुसर्‍या ओळीने आवृत्ती क्रमांक दर्शविला पाहिजे. आवृत्ती 2 आणि 3 प्लग-इन अधिक विजेट्सना समर्थन देतात, परंतु आवृत्ती 3 प्लग-इन ओड्यासिटी 1.3.3 किंवा पूर्वीच्या आवृत्तीत समर्थित नाहीत. पुढील ओळ प्लग-इनचा प्रकार आहे, ज्याची चर्चा खाली केली आहे. त्यानंतर प्लग-इनचे नाव येते, जे मेनू बारमध्ये प्रदर्शित होते आणि नंतर "actionक्शन" होते, जे प्लग-इन प्रक्रियेत व्यस्त असताना ओड्यासिटी प्रदर्शित होते. याशिवाय इतर पर्यायी रेषा देखील आहेत. अर्धविराम (;) ने प्रारंभ न होणारी कोणतीही ओळ Nyquist कोड असल्याचे गृहित धरली जाते आणि कार्यान्वित केली जाईल.
20 20 Audacity has support for three types of plug-ins that can be written in Nyquist: ऑडीसिटीला तीन प्रकारचे प्लगइनचे समर्थन आहे जे नायक्विस्टमध्ये लिहिले जाऊ शकतात:
21 21 ;type generate ; प्रकार व्युत्पन्न
22 22 ;type process टाइप प्रक्रिया
23 23 ;type analyze टाइप करा
24 24 These correspond to the three menus that can contain plug-ins: Generate, Effect, and Analyze. Generate plug-ins are expected to generate new audio from scratch, Effect plug-ins (“process”) modify existing audio in-place, and Analyze plug-ins process audio but do not modify it (though they are allowed to add labels). हे तीन मेनूशी संबंधित आहेत ज्यात प्लगइन असू शकतात: व्युत्पन्न, प्रभाव आणि विश्लेषण. व्युत्पन्न प्लग-इनने आरंभ वरून नवीन ध्वनी व्युत्पन्न करणे आवश्यक आहे, प्रभावी प्लग-इन ("प्रक्रिया") विद्यमान ध्वनी जागेमध्ये सुधारित करा आणि प्लगइन प्रक्रिया ध्वनीचे विश्लेषण करा परंतु ते सुधारित करु नका (जरी त्यांना लेबले जोडण्याची परवानगी असेल) .
2५ 2५ For Effect and Analyze plug-ins, Audacity sets up the Nyquist environment so that the audio the user has selected is in the variable s. All of the expressions in the plug-in file are executed in order, and the return value of the last expression is substituted for the selection in Audacity. If the last expression does not return audio, Audacity returns an error. प्लग-इन इफेक्ट आणि विश्लेषणासाठी ओड्यासिटी नेयक्विस्ट वातावरण सेट करते जेणेकरुन वापरकर्त्याने निवडलेला ध्वनी व्हेरिएबलमध्ये असेल. प्लग-इन धारिकामधील सर्व अभिव्यक्ती क्रमाने कार्यान्वित केल्या जातात आणि शेवटच्या अभिव्यक्तीचे परतावे मूल्य ओड्यासिटीच्या निवडीसाठी दिले जाते. अंतिम अभिव्यक्ती ध्वनी परत करत नसल्यास ओड्यासिटीने त्रुटी परत केली.

44 44 If Audacity finds at least one “control” line, it will open a dialog to prompt the user for certain parameters to the plug-in. Each parameter consists of a text box and a slider, and after the user has entered each one, the final value will be stored in a Nyquist variable with a name that you specify on the “control” line. Here’s what the dialog for the Delay effect shown above looks like in Audacity for Linux: जर ओड्यासिटीला कमीतकमी एक "नियंत्रण" ओळ आढळली तर ती वापरकर्त्यास प्लग-इनमध्ये काही पॅरामीटर्ससाठी सूचित करण्यासाठी एक संवाद उघडेल. प्रत्येक पॅरामीटरमध्ये मजकूर बॉक्स आणि स्लाइडर असते आणि वापरकर्त्याने प्रत्येक प्रविष्ट केल्यावर अंतिम मूल्य Nyquist व्हेरिएबलमध्ये आपण "नियंत्रण" लाइनवर निर्दिष्ट केलेल्या नावाने संग्रहित केले जाईल. वर दर्शविलेल्या विलंब प्रभावासाठी संवाद हा लिनक्सच्या ओड्यासिटीमध्ये दिसत आहे.

७६ ७६ Where to go from here येथून कुठे जायचे
७७ ७७ Audacity comes with some sample plug-ins that you can examine or modify as a starting point. The best way to learn Nyquist is to try it. If you’re having trouble debugging, consider downloading the standalone version of Nyquist (see the link in Part 1). धृष्टता काही नमुने प्लगइनसह येते जे आपण प्रारंभ बिंदू म्हणून परीक्षण करू किंवा सुधारित करू शकता.
