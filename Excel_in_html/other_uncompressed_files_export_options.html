९ ९ Header शीर्षलेख
१० १० Specifies the "type" or extension of the exported file. निर्यात केलेल्या धारिकाचा "प्रकार" किंवा विस्तार निर्दिष्ट करते.
११ ११ "Raw (header-less)" or "RF६४ (RIFF ६४)" headers (with PCM encoding in either case) may be used if extremely large files similar to WAV are required. The RF६४ size limit is about १६ exabytes, making it suitable for very long multi-channel files. "रॉ (शीर्षलेख-कमी)" किंवा "आरएफ (६४ (आरआयएफएफ W ६४)" हेडर (एकतर पीसीएम एन्कोडिंगसह) वापरला जाऊ शकतो जर डब्ल्यूएव्हीसारख्या अत्यंत मोठ्या धारिका आवश्यक असतील. आरएफ ६४ आकार मर्यादा सुमारे १६ एक्सबाइट्स आहे, यामुळे ती बर्‍याच लांब मल्टी-वाहिनी धारिकांसाठी उपयुक्त आहे.
१२ १२ Encoding एन्कोडिंग
१३ १३ Determines the codec and where appropriate, the sample format (for example, १६-bit) of the audio. The fewer the number of bits, the less accurately the audio waveform will be represented, but the smaller file size that will be required. कोडेक आणि जेथे योग्य असेल तेथे ध्वनीचे नमुना स्वरूप (उदाहरणार्थ, १६-बिट) निर्धारित करते. बिट्सची संख्या जितकी कमी असेल तितकेच अचूकपणे ध्वनी वेव्हफॉर्मचे प्रतिनिधित्व केले जाईल, परंतु आवश्यक असलेल्या लहान धारिकाचा आकार.
१४ १४ Only the most commonly used encoding types are shown below. Those that use less than १६-bit format (or are lossy codecs like GSM) are usually optimized for specific purposes such as speech telecommunication. With the possible exception of (A)DPCM, higher quality audio for the same reduced file size can generally be obtained by using other lossy codecs like MP३, OGG or AAC. These use psychoacoustic modelling to try and make the retained audio sound similar to the original. केवळ सर्वात सामान्यपणे वापरल्या जाणार्‍या एन्कोडिंग प्रकार खाली दर्शविले आहेत. जे १६-बिटपेक्षा कमी स्वरूपात वापरतात (किंवा जीएसएमसारखे हानीकारक कोडेक्स आहेत) सामान्यत: स्पीच टेलिकम्युनिकेशनसारख्या विशिष्ट हेतूंसाठी अनुकूलित केले जातात. (ए) डीपीसीएमच्या संभाव्य अपवादाने, समान धारिका आकारासाठी उच्च प्रतीचे ध्वनी सहसा एमपी ३, ओजीजी किंवा एएसी सारख्या इतर हानीकारक कोडेक्सचा वापर करून मिळवता येऊ शकतात. हे मूळसारखेच राखलेले ध्वनी ध्वनी वापरुन पहाण्यासाठी आणि बनविण्यासाठी साइकोएकॉस्टिक मॉडेलिंगचा वापर करतात.
१५ १५ Signed/unsigned PCM: These formats represent the audio data by integer numbers. There are ८-bit, १६-bit, २४-bit, or ३२-bit choices. साइन इन / स्वाक्षरीकृत पीसीएम: हे स्वरूप पूर्णांक संख्येद्वारे ध्वनी डेटाचे प्रतिनिधित्व करतात. ८-बिट, १६-बिट, २४-बिट किंवा ३२-बिट निवडी आहेत.
१६ १६ Float: These formats represent the audio data by floating-point numbers. This allows a much wider range of sample values to be represented, further increasing its accuracy. There are "३२-bit" and "६४-bit" choices. फ्लोट: हे स्वरूप फ्लोटिंग पॉइंट नंबरद्वारे ध्वनी डेटाचे प्रतिनिधित्व करतात. हे नमुना मूल्यांच्या विस्तृत रूढीचे प्रतिनिधित्व करण्यास अनुमती देते आणि आणखी त्याची अचूकता वाढवते. येथे "३२-बिट" आणि "६४-बिट" निवडी आहेत.
१७ १७ U-Law, A-Law: ८-bit formats using different types of companding algorithms. These reduce the dynamic range of the audio when written to the file, but restore it when played back. यू-लॉ, ए-लॉः विविध प्रकारचे कंपॅंडिंग अल्गोरिदम वापरुन ८-बीट स्वरूप. हे धारिकावर लिहिताना ध्वनीची डायनॅमिक श्रेणी कमी करते, परंतु परत प्ले झाल्यावर ते पुनर्संचयित करते.
१८ १८ ADPCM, DPCM: ADPCM are ४-bit formats that sound near-identical to PCM except where there is a lot of low frequency content. There are various ADPCM choices according to the header chosen, such as IMA, Microsoft, G७२१, G७२३ and VOX. ADPCM and DPCM both save storage space by predicting the next sample, and encoding the PCM values only as differences between the predicted and actual value. ADPCM saves further space by varying the size of the quantization step. एडीपीसीएम, डीपीसीएमः एडीपीसीएम हे ४-बिट स्वरूप आहेत जे पीसीएम जवळ समान दिसतात जिथे बरेच कमी वारंवारता सामग्री आहे. निवडलेल्या शीर्षलेखानुसार विविध एडीपीसीएम निवडी आहेत, जसे की आयएमए, मायक्रोसॉफ्ट, जी ७२१, जी ७२३ आणि व्हीओएक्स. एडीपीसीएम आणि डीपीसीएम दोघेही पुढील नमुन्याचा अंदाज लावून स्टोरेज स्पेस वाचवतात, आणि पीसीएम व्हॅल्यूज फक्त पूर्वानुमानित आणि वास्तविक मूल्यामधील फरक म्हणून एन्कोड करतात. एडीपीसीएम परिमाण चरणच्या आकारात बदल करून पुढील जागा वाचवते.
१९ १९ GSM ६.१०: A lossy voice codec producing small file sizes, used on GSM mobile digital telephone networks. जीएसएम १०.१०: जीएसएम मोबाइल डिजिटल टेलिफोन नेटवर्कवर वापरला जाणारा लहान धारिका आकार देणारा तोट्याचा व्हॉईस कोडेक.
