८ ८ When importing audio files ध्वनी धारिका आयात करताना
९ ९ Audacity always makes a copy of compressed files like MP३, OGG and FLAC when it imports them. The data is copied either into the temporary folder, or into the _data folder of an open project. Uncompressed files (WAV or AIFF) are very much larger, so may lead to a shortage of disk space if copied in. Audacity therefore allows you to choose from the following two options when importing WAV or AIFF files: ओड्यासिटी नेहमी एमपी ३, ओजीजी आणि एफएलएसी सारख्या संकुचित धारिकांची एक प्रत बनविते जेव्हा ती आयात करते. एकतर तात्पुरते निर्देशिकामध्ये किंवा ओपन प्रकल्पच्या _डेटा निर्देशिकामध्ये डेटा नक्कल केला जातो. अनकम्प्रेस्ड धारिका्स (डब्ल्यूएव्ही किंवा एआयएफएफ) खूपच मोठी आहेत, त्यामुळे नक्कल केली असल्यास डिस्क स्पेसची कमतरता उद्भवू शकते. म्हणून डब्ल्यूएव्ही किंवा एआयएफएफ धारिका्स आयात करताना ओड्यासिटी आपल्याला खालील दोन पर्यायांमधून निवडण्याची परवानगी देते.
१0 १0 Copy uncompressed files into the project (safer): Audacity makes a copy of the imported file(s), so you can then move, rename or delete them if you wish. Also choose this option if you need to open your Audacity project on another computer, or send it to someone else. Note: you do not need to choose this option if you are sending an exported audio file like an MP३ to someone else. प्रकल्पमध्ये संकुचित धारिका नक्कल करा (अधिक सुरक्षित): धृष्टता आयात केलेल्या धारिकांची नक्कल बनवते, जेणेकरुन आपण नंतर इच्छित असल्यास त्यास हलवू, नाव बदलू किंवा हटवू शकता. आपल्याला आपला ओड्यासिटी प्रकल्प दुसर्‍या संगणकावर उघडण्याची किंवा तो एखाद्यास पाठविण्याची आवश्यकता असल्यास हा पर्याय देखील निवडा. टीप: आपण एमपी ३ सारखी निर्यात केलेली ध्वनी धारिका एखाद्यास दुसर्‍यास पाठवत असल्यास आपल्याला हा पर्याय निवडण्याची आवश्यकता नाही.
११ ११ Read uncompressed audio files from original location (faster): Audacity reads the imported files from the computer folder you first imported them from, without copying them in. Therefore with this option set, you must not move, rename or delete imported uncompressed files, or the folder they are in. The advantage of this option is that long files can be imported in only a few seconds, using On-Demand Loading. मूळ स्थानावरून (वेगवान) संकुचित ध्वनी धारिका वाचा: ओड्यासिटी आपण ज्या संगणक निर्देशिकामधून प्रथम आयात केल्या त्या धारिका्सची नक्कल न करता वाचतात. म्हणूनच या पर्याय सेटसह, आपण आयातित असम्पीडित धारिका हलवू, पुनर्नामित किंवा हटवू नयेत किंवा या निर्देशिकामध्ये ते आहेत. या पर्यायाचा फायदा म्हणजे ऑन-डिमांड लोडिंगचा वापर करुन काही धारिका फक्त काही सेकंदातच धारिका आयात केल्या जाऊ शकतात.

१४ १४ Normalize all tracks in project: The imported file (in any format) and all audio tracks already on screen will be normalized to your current setting for the Normalize effect. प्रकल्पातील सर्व गीतपट्टा सामान्य करा: सामान्याइझ प्रभावासाठी आयात केलेली धारिका (कोणत्याही स्वरूपात) आणि आधीपासून स्क्रीनवर असलेले सर्व ध्वनी गीतपट्टा आपल्या वर्तमान समयोजनमध्ये सामान्य केले जातील.
१५ १५ When exporting tracks to an audio file ध्वनी धारिकावर गीतपट्टा निर्यात करताना
१६ १६ Mix down to Stereo or Mono: This is Audacity's default behavior and means that if your project contains only mono tracks panned to center, they will be mixed upon export to a file असलेले a single mono channel. The audio will still play equally out of both speakers on most systems. If you have any stereo tracks in your project (or any mono tracks panned away from center) the exported file will be stereo, असलेले two channels (left and right). स्टीरिओ किंवा मोनोमध्ये मिसळा: हे ओड्यासिटीचे पूर्वनिर्धारित वर्तन आहे आणि याचा अर्थ असा की आपल्या प्रकल्पात मध्यभागी पॅन केलेले मोनो गीतपट्टा असतील तर ते एकाच मोनो वाहिनी असलेल्या धारिकामध्ये निर्यात केल्यावर मिसळले जातील. बर्‍याच उपकरणवरील दोन्ही स्पीकर्सपैकी ध्वनी अद्याप तितकाच प्ले होईल. आपल्याकडे आपल्या प्रकल्पात कोणतेही स्टीरिओ गीतपट्टा असल्यास (किंवा कोणत्याही मोनो गीतपट्टा मध्यभागी दूर पॅन केलेले असल्यास) निर्यात केलेली धारिका दोन वाहिनी (डावी आणि उजवी) असलेली स्टिरीओ असेल.
१७ १७ Use custom mix: When this option is enabled, an Advanced Mixing Options window appears immediately after completing the Export Audio Dialog (custom mix is not available when using Export Multiple or Macros). Advanced Mixing gives you much greater flexibility in mapping your audio tracks to specific output channels in the exported file. In particular it allows export of multi-channel files (that is, having more than two channels of audio) in WAV, AIFF, OGG, FLAC, AC३ and M४A formats. Any leading white space between time zero and the start of the audio will be rendered as silence in the exported channels, preserving their time offset. सानुकूल मिक्स वापरा: जेव्हा हा पर्याय सक्षम केला असेल तेव्हा निर्यातध्वनी संवाद पूर्ण केल्यावर प्रगत मिक्सिंग ऑप्शन्स विंडो लगेच दिसून येतो (निर्यातमल्टीपल किंवा मॅक्रो वापरताना सानुकूल मिक्स उपलब्ध नाही). प्रगत मिश्रण आपल्याला निर्यात केलेल्या धारिकामधील विशिष्टबाहेरपुट वाहिनीवर आपले ध्वनी गीतपट्टा मॅप करण्यात बरेच लवचिकता देते. विशेषत: ते डब्ल्यूएव्ही, एआयएफएफ, ओजीजी, एफएलएसी, एसी ३ आणि एम ४ ए स्वरूपात मल्टी-वाहिनी धारिका (म्हणजेच दोनपेक्षा जास्त ध्वनीच्या वाहिनी असलेल्या) च्या निर्यात करण्यास परवानगी देते. वेळ शून्य आणि ध्वनीच्या सुरूवातीच्या दरम्यानची कोणतीही अग्रगण्य जागा निर्यातवाहिनीमध्ये मौन म्हणून प्रस्तुत केली जाईल, त्यांचा वेळ ऑफसेट जतन करुन ठेवली जाईल.
१८ १८ Show Metadata Tags editor before export: निर्यात करण्यापूर्वी मेटाडेटा टॅग संपादक दर्शवा:

2६ 2६ When using File > Export > Export MIDI you can export to .mid or .gro extension. The .gro extension signifies the Allegro file format which exports the Audacity Note Track as human-readable plain text. This preference controls whether the .gro file represents time and duration in seconds (default) or beats. धारिका> निर्यात> निर्यातएमआयडी वापरताना आपण .mid किंवा .gro एक्सटेंशनमध्ये निर्यातकरू शकता. .Gro विस्तार अ‍ॅलेग्रो धारिका स्वरूपनास सूचित करतो जो ओड्यासिटी नोट गीतपट्टाला मानव-वाचन करण्यायोग्य साधा मजकूर म्हणून निर्यात करतो. हे प्राधान्य .gro धारिका सेकंदात (पूर्वनिर्धारित) किंवा बीट्समध्ये वेळ आणि कालावधी दर्शवते की नाही हे नियंत्रित करते.
