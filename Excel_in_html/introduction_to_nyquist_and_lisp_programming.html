४ ४ Nyquist was written by Roger B. Dannenberg and was intended to be used as a complete programming language for audio synthesis and analysis, with support for MIDI, audio recording and playback, file I/O, object-oriented programming, profiling, debugging and more. Audacity uses only a subset of Nyquist’s functionality, allowing you to take simple Nyquist functions and use them to process audio data. Audacity doesn’t include any support for debugging Nyquist code, so if you are trying to write a complicated plug-in, you may find it easier to get the full version of Nyquist and develop there, then turn it into an Audacity plug-in. Nyquist is available from the Carnegie Mellon University Computer Music Project. नायक्विस्ट रॉजर बी. डॅन्नेनबर्ग यांनी लिहिलेले होते आणि ध्वनी संश्लेषण आणि विश्लेषणासाठी एमआयडीआय, ध्वनी ध्वनीमुद्रणिंग आणि प्लेबॅक, धारिका आय / ओ, ऑब्जेक्ट-ओरिएंटेड प्रोग्रामिंग, प्रोधारिकािंग, डीबगिंग आणि बरेच काही यासाठी एक संपूर्ण प्रोग्रामिंग भाषा म्हणून वापरली जाण्याची इच्छा होती . ओड्यासिटी केवळ Nyquist च्या कार्यक्षमतेचा उपसंच वापरते, आपल्याला साधी Nyquist कार्ये करण्याची आणि ध्वनी डेटावर प्रक्रिया करण्यासाठी त्यांचा वापर करण्याची परवानगी देते. ओड्यासिटी मध्ये एनक्यूइस्ट कोड डीबग करण्यासाठी कोणतेही समर्थन समाविष्ट नाही, म्हणून जर आपण एक गुंतागुंतीचे प्लग-इन लिहिण्याचा प्रयत्न करीत असाल तर आपल्याला Nyquist ची संपूर्ण आवृत्ती मिळवणे आणि तेथे विकसित करणे सुलभ वाटेल, तर त्यास ओड्यासिटी प्लग-इनमध्ये रूपांतरित करा. . कार्केगी मेलॉन युनिव्हर्सिटी संगणक संगीत प्रकल्पातून Nyquist उपलब्ध आहे.

१६ १६ Nyquist allows you to use the Nyquist programming language to write your own plug-in effects for Audacity. Unlike VST and LADSPA plug-ins, Nyquist plug-ins can be written using an ordinary text editor and don’t need to be compiled. ऑयडीसिटी आपल्याला स्वतःचा प्लग-इन प्रभाव लिहिण्यासाठी Nyquist प्रोग्रामिंग भाषा वापरण्याची परवानगी देतो. व्हीएसटी आणि एलएडीएसपीए प्लगइनच्या विपरीत, न्यक्विस्ट प्लगइन एक सामान्य मजकूर संपादक वापरून लिहिले जाऊ शकतात आणि त्यास कंपाईल करण्याची आवश्यकता नाही.

२१ २१ Lisp लिस्प
२२ २२ Nyquist is based on Lisp. If you have programmed in Lisp before, you can skim this section or go directly to the next page: Programming in Nyquist . Otherwise, here’s an extremely brief introduction to Lisp: Nyquist लिस्पवर आधारित आहे. जर आपण आधी लिस्पमध्ये प्रोग्राम केलेले असेल तर आपण हा विभाग वगळू शकता किंवा पुढील पृष्ठावर जाऊ शकताः प्रोग्रामिंग नायक्विस्टमध्ये. अन्यथा, इथे लिस्पची अत्यंत थोडक्यात माहिती आहेः
२३ २३ In Lisp (and therefore Nyquist), everything is an S-Expression, which is just a list of tokens (words) separated by whitespace and enclosed in parentheses. The name of the function is always the first token in an S-Expression, and all of the other tokens are arguments to this function. Here’s a simple example: लिस्प (आणि म्हणून न्यक्विस्ट) मध्ये, प्रत्येक गोष्ट एस-एक्सप्रेशन असते, जी केवळ स्पेसद्वारे विभक्त आणि कंसात बंद केलेल्या टोकनची (शब्द) यादी आहे. एस-एक्सप्रेशन मधील फंक्शनचे नाव नेहमीच प्रथम टोकन असते आणि इतर सर्व टोकन या फंक्शनचे वितर्क असतात. येथे एक साधे उदाहरण आहेः
२४ २४ (setf area (* ३.१४१५९ (expt radius २))) (सेटफ क्षेत्र (* ३.१४१५९ (एक्सप्ट त्रिज्या २)))
२५ २५ Let’s break down this example. The outermost S-expression has three members. The first one, setf, is the name of the function (it stands for set-field). setf is used to assign a value to a variable. (There are other similar functions, like set and setq, but setf is the most powerful, so it’s the one we’ll use in our examples.) After setf comes area, which is the name of the variable we’re going to set. Next comes the value to assign to this variable, which in this case is another S-expression. चला या उदाहरणाचे खंडन करूया. बाहेरील एस-अभिव्यक्तीचे तीन सदस्य आहेत. पहिला सेटफ, फंक्शनचे नाव आहे (ते सेट-फील्डसाठी वापरले जाते). व्हेरिएबलला व्हॅल्यू देण्यासाठी सेटफचा वापर केला जातो. (इतर समान कार्ये आहेत जसे की सेट आणि सेटेक, परंतु सेटफ सर्वात शक्तिशाली आहे, म्हणूनच आपण आपल्या उदाहरणांमध्ये वापरू.) सेटफचे क्षेत्रफळ आल्यानंतर आपण बदलणार्या व्हेरिएबलचे नाव आहे. . पुढे या व्हेरिएबलला असाइन करण्यासाठी व्हॅल्यू मिळेल, जे या प्रकरणात आणखी एक एस-एक्सप्रेशन आहे.
२६ २६ Lisp doesn’t have any special operators for Math functions – they’re all functions like everything else, using prefix notation, where the name of the function (or operator) comes before its arguments. So instead of ३*७ for the product of ३ and ७, in Lisp you would write (* ३ ७). In Nyquist, the expt (exponent) function raises its first argument to the power of the second argument. Therefore (* ३.१४१५९ (expt radius २)) means ३.१४१५९ times the square of radius, or the formula for the area of a circle. लिस्पकडे मॅथ फंक्शन्ससाठी कोणतेही विशेष ऑपरेटर नसतात - ते प्रत्येकाच्या सारख्या सर्व फंक्शन्स असतात, प्रत्यय नोटेशन वापरुन, जेथे फंक्शनचे नाव (किंवा ऑपरेटर) त्याच्या वितर्कांपूर्वी येते. तर ३ आणि ७ च्या उत्पादनाऐवजी लिस्पमध्ये (* ३ ७) लिहा. Nyquist मध्ये, एक्सप्ट (घातांक) फंक्शन दुसर्‍या वितर्क च्या सामर्थ्यावर आपला पहिला युक्तिवाद दर्शवितो. म्हणून (* १४.११४११५ ((एक्सप्ट रेडियस २)) म्हणजे त्रिज्याच्या चौकोनाच्या पटापेक्षा जास्त किंवा वर्तुळाच्या क्षेत्राचे सूत्र.
२७ २७ Rather than typing in this full expression every time, let’s define a function for the area of the circle, that we can call every time we need it: प्रत्येक वेळी या पूर्ण अभिव्यक्तीवर टाइप करण्याऐवजी आपण वर्तुळाच्या क्षेत्रासाठी फंक्शन परिभाषित करू या, ज्यावेळेस आम्हाला जेव्हा आवश्यक असेल तेव्हा आम्ही कॉल करू:
२८ २८ (defun circlearea (radius) (सर्कलिया (त्रिज्या)
२९ २९ (* ३.१४१५९ (expt radius २))) (* ३.१४१५९ (एक्सपोर्ट त्रिज्या २%)))
३० ३० The defun function is used to define a new function. The first argument is the name of the function, in this case circlearea. The second argument is a list of arguments to the function to be defined – this is one of the few cases where you have an S-expression that is not interpreted as a function call. Finally the last expression is the value of the function. Now if we want to compute the area of a circle of radius r, we just need to compute: Defun फंक्शन नवीन फंक्शन परिभाषित करण्यासाठी वापरले जाते. प्रथम युक्तिवाद फंक्शनचे नाव आहे, या प्रकरणात मंडळामध्ये. दुसरा युक्तिवाद म्हणजे कार्य करण्यासाठी परिभाषित केलेल्या वितर्कांची यादी - हे अशा काही प्रकरणांपैकी एक आहे जेथे आपल्याकडे एस-एक्सप्रेशन आहे ज्याचे फंक्शन कॉल म्हणून अर्थ लावले जात नाही. शेवटी अंतिम अभिव्यक्ती फंक्शनचे मूल्य आहे. जर आपल्याला त्रिज्या r च्या वर्तुळाचे क्षेत्रफळ मोजायचे असेल तर आपल्याला फक्त गणना करणे आवश्यक आहेः
३१ ३१ (setf area (circlearea r)) (सेटफ क्षेत्र (वर्तुळ क्षेत्र आर))

So the second element of the list is (first (rest mylist)). गोष्टी यादीतून काढून टाकण्यासाठी आपण प्रथम आणि उर्वरित कार्ये वापरू शकता. (पारंपारिकपणे, त्यांना अनुक्रमे कार आणि सीडीआर म्हटले गेले, परंतु बुफर्स्ट आणि विश्रांती लक्षात ठेवणे खूप सोपे आहे. दोन्ही नावे सेट्स नेक्विस्टमध्ये समर्थित आहेत.) (प्रथम मायलिस्ट) आउटपुट १ आहे, आणि उर्वरित मायलिस्टचे आउटपुट आहे यादी (२ ३ ४ ५). तर सूचीचा दुसरा घटक म्हणजे (प्रथम (बाकीची माझी यादी)).
