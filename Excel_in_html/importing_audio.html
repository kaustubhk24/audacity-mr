५५ ५५ Audio files can carry descriptive information in a metadata container, this allows information such as the title, artist, album, track number, and other information about the file to be stored in the audio file itself. ध्वनी धारिका मेटाडाटा कंटेनरमध्ये वर्णनात्मक माहिती ठेवू शकतात, यामुळे शीर्षक, कलाकार, अल्बम, गीतपट्टा क्रमांक आणि धारिकाविषयीची अन्य माहिती ध्वनी धारिकामध्येच संग्रहित केली जाऊ शकते.
५६ ५६ Each time an audio file is imported into the same project window, the metadata tags for that file replace the previously stored metadata for the project. So if the last imported file has no metadata, the project will no longer have any stored metadata. Using one of the above methods that imports each file into its own project window will ensure each file's metadata is stored. प्रत्येक वेळी त्याच प्रकल्प विंडोमध्ये एखादी ध्वनी धारिका आयात केली जाते तेव्हा त्या धारिकासाठी मेटाडेटा टॅग प्रकल्पासाठी पूर्वी संग्रहित मेटाडेटा पुनर्स्थित करतात. तर शेवटच्या आयातीत धारिकामध्ये मेटाडेटा नसल्यास, यापुढे यापुढे कोणताही संचयित मेटाडेटा नसेल. वरीलपैकी एक पद्धत वापरली जी प्रत्येक धारिकाला त्याच्या स्वतःच्या प्रकल्प विंडोमध्ये आयात केली तर प्रत्येक धारिकाचा मेटाडेटा संचयित होईल याची खात्री होईल.
५७ ५७ Using the file type dropdown menu in the Open and Import Audio Dialogs ओपन आणि इम्पोर्ट ध्वनी डायलॉग मधील धारिका टाइप ड्रॉपडाउन मेनू वापरणे
५८ ५८ This menu (called "Format" on Mac) contains a list of different file types. या मेनूमध्ये (मॅकवरील "स्वरूप" म्हटले जाते) विविध धारिका प्रकारांची सूची आहे.
५९ ५९ Choosing a particular file type in the menu performs two functions. मेनूमध्ये विशिष्ट धारिका प्रकार निवडणे दोन कार्ये करते.
६० ६० The file type choice filters the list of files in the window by restricting it to files of that type. धारिका प्रकारची निवड विंडोमधील धारिका्सची सूची त्या प्रकारच्या धारिका्सपुरती मर्यादित ठेवून गाळणी करते.
६१ ६१ The file type choice determines which importer attempts the file first when using the Open or Import Audio dialogs. For example, a WAV file could be imported by Audacity's native WAV importer or by the optional FFmpeg library if this was installed. धारिका प्रकार निवड ओपन किंवा इम्पोर्ट ध्वनी संवाद वापरताना कोणता आयातकर्ता प्रथम धारिकाचा प्रयत्न करतो हे ठरवते. उदाहरणार्थ, डब्ल्यूएव्ही धारिका ओड्यासिटीच्या मुळ डब्ल्यूएव्ही आयातकाद्वारे किंवा ही स्थापित केली असल्यास पर्यायी एफएफएमपीजी लायब्ररीद्वारे आयात केली जाऊ शकते.
६2 ६2 Additionally, rules for the order in which different importers attempt files of particular types can be created at Extended Import Preferences with an option to over-ride the Open and Import Audio file type choice. See Import Filtering and Importer Order for more details. या व्यतिरिक्त, ओपन अँड इम्पोर्ट ध्वनी धारिका प्रकार निवडीवर जादा-राइडिंग पर्याय असलेल्या विस्तारित आयात प्राधान्ये येथे विविध आयातकर्त्यांनी विशिष्ट प्रकारच्या धारिकांचा प्रयत्न केल्याच्या ऑर्डरचे नियम तयार केले जाऊ शकतात. अधिक तपशीलांसाठी आयात गाळणीिंग आणि आयातकर्ता ऑर्डर पहा.
६३ ६३ Importing Uncompressed Audio Files संकुचित ध्वनी धारिका आयात करीत आहे
६४ ६४ The two most common uncompressed file types you will encounter are WAV and AIFF. दोन सामान्य कॉम्प्रेसप्रेस धारिका प्रकार आपल्यास आढळतील ते म्हणजे डब्ल्यूएव्ही आणि एआयएफएफ.
६५ ६५ Warning Dialog when Importing Uncompressed Audio Files संकुचित ध्वनी धारिका आयात करताना चेतावणी संवाद
६६ ६६ The first time you import an uncompressed audio file into Audacity, you will see this dialog. प्रथमच आपण असुरक्षित ध्वनी धारिका ओड्यासिटीमध्ये आयात करता तेव्हा आपल्याला हा संवाद दिसेल.
६७ ६७ For each imported file, you can choose: प्रत्येक आयात केलेल्या धारिकासाठी आपण हे निवडू शकता:
६८ ६८ Whether to copy the imported file's data into the project or to read its data directly from the file, as described below. प्रकल्पमध्ये आयात केलेल्या धारिकाचा डेटा कॉपी करायचा किंवा खाली वर्णन केल्यानुसार त्याचा डेटा थेट धारिका्समधून वाचायचा.
६९ ६९ Whether to make your copy or read directly choice permanent and never show this warning again. आपली प्रत बनवायची की थेट निवड कायम वाचली पाहिजे आणि या चेतावणी पुन्हा कधीही दर्शवू नका.
७० ७० Import Methods आयात पद्धती
७१ ७१ Make a copy before editing (safer) संपादन करण्यापूर्वी एक प्रत बनवा (अधिक सुरक्षित)
७2 ७2 This is the default setting. When this option is chosen, Audacity copies the imported file(s) into the project's audio data. It is essential that files are copied into the project if the Audacity project is to be moved, opened on another computer or sent to someone else, or if you need to to move, rename or delete the original files. ही पूर्वनिर्धारित सेटिंग आहे. जेव्हा हा पर्याय निवडला जातो तेव्हा ओड्यासिटी आयात केलेल्या धारिका (ओं) प्रकल्पच्या ध्वनी डेटामध्ये कॉपी करते. ओड्यासिटी प्रकल्प हलविणे, दुसर्‍या संगणकावर उघडणे किंवा दुसर्‍या एखाद्याला पाठविणे किंवा धारिका हलविणे, नाव बदलणे किंवा मूळ धारिका्स हटविणे आवश्यक असल्यास त्या प्रकल्पात धारिका्सची प्रत बनवणे आवश्यक आहे.
७३ ७३ This option does not make a backup copy of the original file in its original location. If you do not want to lose the original content of the file when exporting to the same file name and location, please change the file name when exporting. हा पर्याय मूळ धारिकाच्या मूळ स्थानावरील बॅकअप प्रत बनवित नाही. त्याच धारिकाच्या नावावर आणि स्थानावर निर्यात करताना तुम्हाला धारिकाची मूळ सामग्री गमवायची नसेल, तर कृपया निर्यात करताना धारिकाचे नाव बदला.
७४ ७४ Read directly from the original (faster) मूळ (वेगवान) वरून थेट वाचा
७५ ७५ When this option is chosen Audacity always reads imported files from the folder you first imported them from, without copying them into the project. Therefore with this option set, you must not move, rename or delete imported uncompressed files, or the folder they are in. The advantage of this option is that you can begin editing a long file almost immediately, since On-Demand Loading will be used to import the file. जेव्हा हा पर्याय निवडला जातो तेव्हा ओड्यासिटी नेहमी आयात केलेल्या धारिका आपण त्यामधून प्रकल्पमध्ये कॉपी केल्याशिवाय त्या त्या आयात केल्या त्या फोल्डरमधून नेहमीच वाचतात. म्हणूनच या ऑप्शन सेटसह, आपण आयातित कंप्रप्रेस धारिका किंवा त्यांचे असलेले फोल्डर हलवू नये, नाव बदलू किंवा हटवू नये. या पर्यायाचा फायदा म्हणजे आपण जवळजवळ त्वरित एक लांब धारिका संपादित करणे सुरू करू शकता, कारण ऑन-डिमांड लोडिंग वापरली जाईल धारिका आयात करण्यासाठी.
७६ ७६ If you export to the same file name and location as the original file, Audacity will make a copy of the original file's content (renamed to include "-old" and a number) and will then read the project from this "-old" file. Therefore you must not move, rename or delete any "-old" files until you have completely finished working on the project. आपण त्याच धारिकाचे नाव आणि मूळ धारिकाच्या स्थानावर निर्यात केल्यास ओड्यासिटी मूळ धारिकाच्या सामग्रीची एक प्रत तयार करेल ("-ल्ड" आणि एक नंबर समाविष्ट करण्यासाठी पुनर्नामित केले जाईल) आणि नंतर या "-ल्ड" धारिकामधून प्रकल्प वाचेल . आपण प्रकल्पवर काम पूर्ण करेपर्यंत आपण कोणतीही "-લ્ડ" धारिका हलवू, पुनर्नामित करू किंवा हटवू नका.
७७ ७७ You can change your import preference at any time in Import / Export Preferences, and you can turn the warning dialog on and off in Warnings Preferences. आपण आयात / निर्यात प्राधान्ये कोणत्याही वेळी आपली आयात प्राधान्ये बदलू शकता आणि आपण चेतावणी प्राधान्यांमध्ये चेतावणी संवाद चालू आणि बंद करू शकता.
७८ ७८ Dependencies अवलंबित्व

At any time you can check which files your project depends on by clicking on Help > Diagnostics > Check Dependencies.... See the Dependencies Dialog page for details. आपण "थेट वाचन करा" पर्याय वापरून आपल्या प्रकल्पात कोणतीही संकुचित ध्वनी धारिका आयात केली तर आपला प्रकल्प त्या धारिकांवर अवलंबून असेल. मदत> डायग्नोस्टिक्स> अवलंबित्व तपासून क्लिक करून आपण आपला प्रकल्प कोणत्या धारिकावर अवलंबून आहे हे आपण कधीही तपासू शकता .... तपशीलांसाठी अवलंबित्व संवाद पृष्ठ पहा.
८० ८० When you save a project you can choose whether or not to copy dependent files into the project. See the Projects Preferences page for details. आपण एखादा प्रकल्प जतन करताना आपण प्रकल्पात अवलंबून असलेल्या धारिका कॉपी कराव्यात की नाही ते निवडू शकता. तपशीलांसाठी प्रकल्प प्राधान्ये पृष्ठ पहा.
८१ ८१ Importing Compressed Audio Files संकुचित ध्वनी धारिका आयात करीत आहे
८2 ८2 When you import a compressed audio file into your project on Windows or Mac, either a lossy format like MP३, MP2, M४A or Ogg Vorbis or the lossless compressed FLAC format, there is no choice between "safer" and "faster" - the file's data is always copied into the project. जेव्हा आपण आपल्या प्रकल्पमध्ये विंडोज किंवा मॅकवर संकुचित ध्वनी धारिका आयात करता, एमपी ३, एमपी 2, एम ४ ए किंवा ओग व्हॉर्बिस किंवा लॉसलेस कॉम्प्रेस केलेले एफएलएसी स्वरूप यासारखे हानिकारक स्वरूप असते, तेव्हा धारिकाचे "सुरक्षित" आणि "वेगवान" दरम्यान पर्याय नसतो. डेटा नेहमी प्रकल्पात कॉपी केला जातो.
८३ ८३ However by default on Linux, FLAC is imported by the Audacity importer for uncompressed files, so the warning for "Importing uncompressed audio files" will appear with the choice to copy in the audio or read it directly. Reading directly permits faster On-Demand Loading. तथापि लिनक्सवर पूर्वनिर्धारितनुसार, एफएलएसी ओड्यासिटी आयातकर्त्याद्वारे संकुचित केलेल्या धारिकांसाठी आयात केले जाते, म्हणून ध्वनीमध्ये कॉपी करण्यासाठी किंवा थेट वाचण्यासाठीच्या निवडीसह "कॉम्प्रेसप्रेस ध्वनी धारिका आयात करणे" ही चेतावणी दिसेल. वाचन थेट मागणीनुसार लोड करण्यास द्रुत परवानगी देते.
८४ ८४ Re-encoding lossy compressed audio filesIf you import a lossy compressed audio file into Audacity, edit it then export it as a lossy compressed audio file, you will lose quality twice, once in the original encoding of the imported audio, then again when you export it from Audacity as a compressed file. So never do production work in lossy compressed audio formats, instead work with a lossless format such as WAV, AIFF or FLAC and for final use then export to compressed formats. लॉसी कॉम्प्रेस केलेले ध्वनी धारिका्सचे पुन्हा एन्कोडिंग ओड्यासिटीमध्ये आपण हानीकारक कॉम्प्रेस केलेल्या ध्वनी धारिकाची आयात करत असल्यास ती संपादित करा नंतर ती हानीकारक संकुचित ध्वनी धारिका म्हणून निर्यात केल्यास आयात केलेल्या ध्वनीच्या मूळ एन्कोडिंगमध्ये एकदा आपण गुणवत्ता दोनदा गमावाल, नंतर पुन्हा निर्यात कराल तेव्हा ते एक संकुचित धारिका म्हणून ओड्यासिटीपासून आहे. म्हणून हानीकारक कॉम्प्रेस केलेले ध्वनी स्वरूपात उत्पादन कार्य कधीही करू नका, त्याऐवजी डब्ल्यूएव्ही, एआयएफएफ किंवा एफएलएसीसारख्या लॉसलेस फॉरमॅटसह कार्य करा आणि अंतिम वापरासाठी नंतर कॉम्प्रेस केलेल्या स्वरूपात निर्यात करा.
८५ ८५ See this page in the Audacity Wiki for other applications that can be used to directly edit MP३ audio files, thereby avoiding lossy re-encoding. हे पृष्ठ अन्य अनुप्रयोगांसाठी ओड्यासिटी विकीमध्ये पहा जे एमपी ३ ध्वनी धारिका थेट संपादित करण्यासाठी वापरले जाऊ शकतात, ज्यामुळे नुकसानीचे पुन्हा एन्कोडिंग टाळले जाईल.
८६ ८६ Importing audio from CDs सीडी वरून ध्वनी आयात करीत आहे
८७ ८७ Audacity does not contain any function for importing (ripping) audio from CDs. This can, however, be achieved by using other applications to extract the data into a file format Audacity does support. ऑडीसीटीमध्ये सीडी वरुन ध्वनी (रिपिंग) आयात करण्यासाठी कोणतेही कार्य नसते. ओड्यासिटी सपोर्ट केलेल्या धारिका स्वरूपात डेटा काढण्यासाठी अन्य अनुप्रयोगांचा वापर करुन हे साध्य केले जाऊ शकते.

८९ ८९ Sample Rate and Sample Format of imported files नमुना दर आणि आयात केलेल्या धारिकांचे नमुना स्वरूप
९० ९० Audacity always imports files at their original sample rate, as displayed in the Track Information Area of the Track Control Panel. गीतपट्टा कंट्रोल पॅनेलच्या गीतपट्टा माहिती क्षेत्रामध्ये दर्शविल्यानुसार ओड्यासिटी नेहमी त्यांच्या मूळ नमुन्या दरावर धारिका आयात करते.
९१ ९१ When importing a file into an empty project window (as always happens when using File > Open... as described below), the Project Rate control in Selection Toolbar changes if necessary to reflect the rate of the file. The Project Rate determines the sample rate a file will be exported at, so no further adjustment is needed to export that file at its original rate. However, once there is already an audio track of any origin in the project (either an imported file, a recording or generated audio), importing an audio file will never change the project rate. रिकाम्या प्रकल्प विंडोमध्ये धारिका आयात करताना (नेहमीप्रमाणेच धारिका> ओपन वापरत असताना असे होते ... खाली वर्णन केल्यानुसार), धारिकाचे दर प्रतिबिंबित करण्यासाठी आवश्यक असल्यास निवड साधनपट्टीमधील प्रकल्प रेट नियंत्रण बदलते. प्रकल्प रेट एक धारिका निर्यात केली जाईल याचा नमुना दर निर्धारित करते, म्हणून त्या धारिकाच्या मूळ दरावर निर्यात करण्यासाठी यापुढे समायोजित करण्याची आवश्यकता नाही. तथापि, एकदा एकदा प्रकल्पमध्ये कोणत्याही मूळचा ध्वनी गीतपट्टा (एकतर आयात केलेली धारिका, ध्वनीमुद्रणिंग किंवा व्युत्पन्न ध्वनी) झाला की ध्वनी धारिका आयात केल्यास प्रकल्प दर कधीही बदलणार नाहीत.
