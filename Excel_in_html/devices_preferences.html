८ ८ Interface मुखपृष्ठ
९ ९ Selects the particular host interface with which Audacity communicates with your chosen playback and recording devices. विशिष्ट होस्ट मुखपृष्ठ निवडतो ज्याद्वारे ओड्यासिटी आपल्या निवडलेल्या प्लेबॅक आणि ध्वनिमुद्रण उपकरणसह संप्रेषण करते.
१० १० Windows: on Windows the choice is between the following host audio interfaces: विंडोजः विंडोजवरील निवड खालील होस्ट ध्वनी मुखपृष्ठ दरम्यान आहे.
११ ११ MME: This is the Audacity default and the most compatible with all audio devices. एमएमईः हे ओड्यासिटी पूर्वनिर्धारित आहे आणि सर्व ध्वनी उपकरणसह सर्वात अनुकूल आहे.
१२ १२ Windows DirectSound: This is more recent than MME with potentially less latency. विंडोज डायरेक्टसाऊंडः संभाव्यत कमी विलंब असलेल्या एमएमईपेक्षा हे अगदी अलीकडील आहे.
१3 १3 Windows WASAPI: This host is the most recent Windows interface between applications (such as Audacity) and the audio interface driver. WASAPI was first officially released in २००७ in Windows Vista. WASAPI is particularly useful for "loopback" recording of computer playback. २4-bit recording devices are supported using this host. Playback is usually emulated. As a result, the playback slider in Mixer Toolbar will only scale the system playback slider's current level up or down rather than directly manipulating that system slider. विंडोज WASAPI : हे होस्ट अनुप्रयोग (जसे की ओड्यासिटी) आणि ध्वनी मुखपृष्ठ ड्राइव्हर दरम्यान सर्वात अलीकडील विंडोज मुखपृष्ठ आहे. WASAPI प्रथम अधिकृतपणे २००७ मध्ये विंडोज व्हिस्टामध्ये प्रसिद्ध झाले. खासकरुन संगणक प्लेबॅक ध्वनिमुद्रणासाठी "लूपबॅक" करण्यासाठी वासा उपयुक्त आहे. हे होस्ट वापरुन २4-बिट ध्वनिमुद्रण उपकरण समर्थित आहेत. प्लेबॅक सहसा अनुकरण केले जाते. याचा परिणाम म्हणून, मिक्सर साधनपट्टीमधील प्लेबॅक स्लाइडर थेट सिस्टम स्लाइडरमध्ये फेरफार करण्याऐवजी सिस्टम प्लेबॅक स्लाइडरची सद्य पातळी खाली किंवा खाली स्केल करेल.
१4 १4 On Windows: विंडोज वर :
१५ १५ Windows DirectSound may by default have only slightly lower latency than MME. पूर्वनिर्धारितनुसार विंडोज डायरेक्टसाऊंडमध्ये एमएमईपेक्षा थोडीशी कमी उशीर असू शकते.
१६ १६ Selecting Windows DirectSound or Windows WASAPI and enabling both "Exclusive Mode" boxes in Windows Sound allows Audacity to request audio direct from the device without resampling. विंडोज डायरेक्टसाऊंड किंवा विंडोज डब्ल्यूएएसपीआय निवडणे आणि विंडोज साऊंडमध्ये दोन्ही "अनन्य मोड" बॉक्स सक्षम केल्याने ओड्यासिटीला रीमॅम्पलिंगशिवाय उपकरणमधून ध्वनी थेट विनंती करण्याची परवानगी मिळते.
१७ १७ Mac: On Mac the only choice is Core Audio. मॅक: मॅकवर केवळ कोअर ध्वनीची निवड आहे.
१८ १८ Linux: On Linux there is often only one option: ALSA. Other options could be OSS and/or Jack Audio Connection Kit (also known as "Jack" or "Jackd"). लिनक्स : लिनक्स वर बहुतेकदा एकच पर्याय असतोः ALSA. इतर पर्याय ओएसएस आणि / किंवा जॅक ध्वनी कनेक्शन किट ("जॅक" किंवा "जॅकड" म्हणून देखील ओळखले जातात) असू शकतात.
१९ १९ Playback प्लेबॅक
२० २० Device: Selects the device used for playback. उपकरण: प्लेबॅकसाठी वापरलेले उपकरण निवडते.
२१ २१ Recording ध्वनिमुद्रण
२२ २२ Device: Selects the device used for recording. उपकरण: ध्वनिमुद्रणसाठी वापरलेले उपकरण निवडते.
२3 २3 On Windows, each entry for recording device consists of the input type (such as microphone), followed by the name of the audio device the input belongs to. If you have multiple audio devices, the list will be grouped so that inputs of each device are grouped together. The input level of the selected device can be adjusted in Mixer Toolbar, but device selection is no longer made there. विंडोज वर, ध्वनिमुद्रण उपकरणसाठी प्रत्येक प्रविष्टीमध्ये इनपुट प्रकार असतो (जसे की मायक्रोफोन), त्यानंतर इनपुट संबंधित ध्वनी उपकरणचे नाव असते. आपल्याकडे एकाधिक ध्वनी उपकरण असल्यास, यादीचे गटबद्ध केले जाईल जेणेकरून प्रत्येक उपकरणची माहिती एकत्रित केली जाईल. निवडलेल्या उपकरणचे इनपुट स्तर मिक्सर साधनपट्टीमध्ये समायोजित केले जाऊ शकते, परंतु उपकरण निवड यापुढे केले जाणार नाही.
२4 २4 Channels: १ (Mono), २ (Stereo) or the number of channels that are provided by the drivers of your sound device. On most inbuilt sound devices, especially on Windows, only mono or stereo will be available. For some devices on Windows, choosing Windows DirectSound in "Host" above may be more likely to reveal options for recording more than two channels. On some devices capable of recording more than two channels, an explicit "multi" device may appear in the "Device" dropdown for recording all the channels simultaneously. वाहिनी : १ (मोनो), २ (स्टीरिओ) किंवा आपल्या आवाज उपकरणच्या ड्रायव्हर्सद्वारे प्रदान केलेल्या वाहिनीची संख्या. बर्‍याच इनबिल्ट साउंड डिव्‍हाइसेसवर, विशेषत: विंडोजवर, फक्त मोनो किंवा स्टीरिओ उपलब्ध असतील. विंडोजवरील काही उपकरणांसाठी, उपरोक्त "होस्ट" मध्ये विंडोज डायरेक्टसाऊंड निवडणे कदाचित दोनपेक्षा जास्त वाहिनी ध्वनिमुद्रण करण्यासाठी पर्याय प्रकट करू शकेल. दोनपेक्षा अधिक वाहिनी ध्वनिमुद्रण करण्यास सक्षम असलेल्या काही उपकरणवर, सर्व वाहिनी एकाच वेळी ध्वनिमुद्रण करण्यासाठी "उपकरण" ड्रॉपडाऊनमध्ये एक स्पष्ट "मल्टी" उपकरण दिसू शकते.
२५ २५ Latency प्रलंबित
२६ २६ Buffer length: A "buffer" is a chunk of audio waiting for the computer to process it. At the default १०० milliseconds (ms) setting, the audio will take ०.१ seconds to travel through the audio interface when recording or playing. Decreasing this value means recordings will be laid down on disk with less latency, and playback may respond faster. However, the computer will have to work faster so that it's ready to process the shorter chunks as soon as required. Setting this value too low (for example to १ ms) will mean the computer will not be able to keep up, and neither recording or playback will work. १०० ms is a safe setting for most computers. बफर लांबी : एक "बफर" ध्वनीचा एक भाग आहे जो संगणकावर प्रक्रिया करण्यासाठी प्रतीक्षा करतो. पूर्वनिर्धारित १०० मिलिसेकंद (एमएस) सेटिंगमध्ये, ध्वनिमुद्रण करताना किंवा प्ले करताना ध्वनीला ध्वनी मुखपृष्ठामधून प्रवास करण्यासाठी ०.१ सेकंद लागतील. हे मूल्य कमी करणे म्हणजे मुखपृष्ठ कमी विलंब असलेल्या डिस्कवर ठेवली जाईल आणि प्लेबॅक वेगवान प्रतिसाद देऊ शकेल. तथापि, संगणकास वेगवान काम करावे लागेल जेणेकरून ते आवश्यकतेनुसार लहान भागांवर प्रक्रिया करण्यास तयार असेल. हे मूल्य खूपच कमी सेट करणे (उदाहरणार्थ १ एमएस पर्यंत) म्हणजे संगणक कायम ठेवण्यास सक्षम होणार नाही आणि ध्वनिमुद्रण किंवा प्लेबॅक देखील कार्य करणार नाही. बहुतेक संगणकांसाठी १०० एमएस ही एक सुरक्षित सेटिंग आहे.
२७ २७ Settings below २०ms, or 3० ms if you are using the Audacity default MME host, may cause you to get clicky playback. जर आपण ओड्यासिटी पूर्वनिर्धारित एमएमई होस्ट वापरत असाल तर २० मि.मी. किंवा ३० एमएस च्या खाली सेटिंग्‍जमुळे आपणास क्‍लेकी प्लेबॅक मिळेल.
२८ २८ Latency compensation: When recording, it takes a small amount of time for the sound to be converted to digital data and written to the track. This time delay is the “recording latency” and is typically around १3० ms (०.१3 seconds) for consumer level equipment. By compensating for this delay, sounds are written to the track at the correct time relative to the Timeline, and other tracks in the project. The actual amount of delay depends on the hardware and audio settings, so for accurate synchronization in multi-track recording, this setting should be calibrated as described in the Latency Test page. प्रलंबित भरपाई : ध्वनी ध्वनिमुद्रण करताना ध्वनी डिजिटल डेटामध्ये रुपांतरित होण्यासाठी आणि गीतपट्टावर लिहिण्यासाठी थोडा वेळ लागतो. यावेळी विलंब म्हणजे “ध्वनिमुद्रण विलंब” आणि ग्राहक पातळीवरील उपकरणासाठी साधारणतः १ १3०० एमएस (०.०3 सेकंद) आहे. या विलंबाची भरपाई करून, वेळलाइनशी संबंधित योग्य वेळी गीतपट्टावर आणि प्रकल्पातील इतर गीतपट्टावर नाद लिहिलेले आहेत. उशीराची वास्तविक मात्रा हार्डवेअर आणि ध्वनी समायोजनवर अवलंबून असते, म्हणूनच अनेक-गीतपट्टावर ध्वनिमुद्रणमध्ये अचूक सिंक्रोनाइझेशनसाठी, प्रलंबित टेस्ट पृष्ठात वर्णन केल्यानुसार ही सेटिंग कॅलिब्रेट केली जावी.
