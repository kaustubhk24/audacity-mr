1६ 1६ This usually means that when reopening the AUP project file normally, Audacity finds an unexpected non-English, accented or control character at the line quoted in the error message. There are two main causes of this. याचा सामान्य अर्थ असा की एओपी प्रकल्प धारिका सामान्यपणे पुन्हा उघडताना ओड्यासिटीला त्रुटी संदेशात उद्धृत केलेल्या ओळीवर अनपेक्षित इंग्रजी, उच्चारण किंवा नियंत्रण वर्ण आढळले. याची दोन प्रमुख कारणे आहेत.
1७ 1७ The project contains imported files whose metadata (like artist or genre) contains control characters. This mostly occurs with iTunes files on Mac where the project was created in an older version of Audacity. प्रकल्पात आयात केलेल्या धारिका आहेत ज्यांचे मेटाडेटा (जसे की कलाकार किंवा शैली) मध्ये नियंत्रण वर्ण आहेत. हे बहुतेक मॅकवरील आयट्यून्स धारिकांसह उद्भवते जिथे ओड्यासिटीच्या जुन्या आवृत्तीमध्ये प्रकल्प तयार केला गेला होता.
1८ 1८ The project was saved containing non-English or accented characters using an old ANSI build of Audacity for Windows. ANSI builds are not properly Unicode aware, so do not support non-English or accented characters. विंडोजसाठी ओड्यासिटीच्या जुन्या एएनएसआय बिल्डचा वापर करून हा प्रकल्प इंग्रजी-नसलेल्या किंवा उच्चारणित वर्णांसह जतन झाला. एएनएसआय बिल्ड्स युनिकोड योग्यरित्या जागरूक नाहीत, म्हणून इंग्रजी-नसलेल्या किंवा उच्चारण केलेल्या वर्णांना समर्थन देऊ नका.
1९ 1९ The latest version of Audacity will always fully support Unicode. You can check your Audacity version at Help > About Audacity... (or Audacity > About Audacity on Mac). ओड्यासिटीची नवीनतम आवृत्ती युनिकोडला पूर्णपणे समर्थन देईल. आपण मदत> ओड्यासिटी बद्दल ... (किंवा ओड्यासिटी> मॅकवरील ओड्यासिटीबद्दल) वर आपली ओड्यासिटी आवृत्ती तपासू शकता.
२0 २0 If Audacity or the computer crashed, Audacity will try to recover the project from its AUTOSAVE file, but the crash may have created a redundant line near the end of the file that triggers the token error. Very occasionally, the AUP file may become corrupted by one or more redundant lines, giving a token error even if there was no apparent crash. To correct this, see remove redundant lines in an AUTOSAVE or AUP file. जर ओड्यासिटी किंवा संगणक क्रॅश झाला असेल तर ओड्यासिटी त्याच्या ऑटोजाव्ह धारिकामधून प्रकल्प पुनर्प्राप्त करण्याचा प्रयत्न करेल, परंतु क्रॅशने धारिकाच्या शेवटी जवळ एक रिडंडंट लाइन तयार केली आहे जी कदाचित टोकन त्रुटीला कारणीभूत करेल. बर्‍याच वेळा, एयूपी धारिका एक किंवा अधिक रिडंडंट लाइनद्वारे दूषित होऊ शकते, स्पष्ट क्रॅश नसतानाही टोकन एरर देऊन. हे दुरुस्त करण्यासाठी, स्वयंपूर्ण किंवा एयूपी धारिकामधील अनावश्यक रेषा काढा.

3२ 3२ Instead, you should always use File > Open... to open an AUP (Audacity Project) file. त्याऐवजी, आपण एयूपी (ओड्यासिटी प्रकल्प) धारिका उघडण्यासाठी नेहमीच धारिका> ओपन ... वापरावे.
33 33 If you see "Error Importing" even when using File > Open..., this usually means the AUP file is seriously damaged, has been encoded other than as UTF-८ without BOM, or only contains NULL (zero) data. आपण धारिका> ओपन ... वापरत असतानाही "आयात करताना त्रुटी" दिसत असल्यास, याचा अर्थ सामान्यत: एयूपी धारिका गंभीरपणे खराब झाली आहे, बीओएमशिवाय यूटीएफ -८ व्यतिरिक्त इतर एन्कोड केली गेली आहे किंवा त्यामध्ये केवळ शून्य (शून्य) डेटा आहे.
3४ 3४ NULL contents शून्य सामग्री
3५ 3५ The contents are NULL and unusable if you open the AUP file in a text editor like Notepad or TextEdit (on Mac) and see empty contents despite the file being a non-zero size. The AUP file is saved last, so a NULL AUP file can occur if you shut down the computer or close the lid on a laptop before the project is completely saved. Be sure to wait until the Audacity Status Bar says the project is saved, or until all project save progress dialogs have closed, before shutting down the machine. आपण नोटपॅड किंवा मजकूर संपादन (मॅक वर) सारख्या मजकूर संपादकात एयूपी धारिका उघडल्यास आणि धारिका शून्य नसलेली आकार असूनही रिक्त सामग्री पाहिल्यास त्यातील सामग्री शून्य आणि निरुपयोगी आहेत. AUP धारिका अखेरची जतन झाली आहे, म्हणून जर आपण संगणक पूर्णपणे बंद करण्यापूर्वी आपण संगणक बंद केल्यास किंवा लॅपटॉपवरील झाकण बंद केल्यास नल एयूपी धारिका येऊ शकते. ओड्यासिटी स्टेटस बार म्हटल्याशिवाय प्रकल्प जतन होईपर्यंत किंवा मशीन बंद करण्यापूर्वी सर्व प्रकल्प जतन प्रगती संवाद बंद होईपर्यंत थांबण्याची खात्री करा.
3६ 3६ If the AUP file appears empty, try to use any backup you created, such a different AUP file for an earlier stage of the project, or an exported WAV file. Failing this, you can try manual recovery of the project's _data folder, but this only works for unedited recordings. जर एयूपी धारिका रिक्त दिसत असेल तर आपण तयार केलेला कोणताही बॅकअप, प्रकल्पाच्या आधीच्या टप्प्यासाठी अशी वेगळी एयूपी धारिका किंवा निर्यात केलेली डब्ल्यूएव्ही धारिका वापरण्याचा प्रयत्न करा. हे अपयशी ठरल्यास आपण प्रकल्पाच्या _डेटा निर्देशिकामध्ये व्यक्तिचलित पुनर्प्राप्तीचा प्रयत्न करू शकता परंतु हे केवळ अप्रशलित ध्वनीमुद्रणिंगसाठी कार्य करते.
3७ 3७ File not saved as UTF-८ without BOM बीओएमशिवाय धारिका यूटीएफ -८ म्हणून जतन केली गेली नाही
3८ 3८ Using Notepad on Windows to save an AUP file containing Unicode characters as UTF-८ will create this problem because Notepad adds a BOM. To solve it, open the file in Notepad++, choose Encoding > Convert to UTF-८ then save the file. Old versions of Notepad ++ may refer to the required command as Convert to UTF-८ without BOM. If the problem occurred after editing the AUP file in TextEdit on Mac, resave the file as text in Brackets or BBEdit. युनिकोड वर्ण असलेली एयूपी धारिका यूटीएफ -८ म्हणून जतन करण्यासाठी विंडोजवर नोटपॅड वापरणे ही समस्या निर्माण करेल कारण नोटपॅडने बीओएम जोडला आहे. त्याचे निराकरण करण्यासाठी, धारिकाला नोटपैड ++ मध्ये उघडा, एन्कोडिंग> यूटीएफ -८ मध्ये रूपांतरित करा आणि नंतर धारिका जतन करा निवडा. नोटपॅड ++ ची जुनी आवृत्ती बीओएमशिवाय यूटीएफ -८ मध्ये रूपांतरित म्हणून आवश्यक आदेशाचा संदर्भ घेऊ शकते. मॅकवरील टेक्स्ट एडिटमध्ये एयूपी धारिका संपादित केल्यानंतर समस्या उद्भवल्यास, ब्रॅकेट्स किंवा बीबीएडिटमध्ये मजकूर म्हणून धारिका पुन्हा जतन करा.

४४ ४४ Error Opening Project: mismatched tag at line x प्रकल्प उघडण्यात त्रुटी: ओळ x वर न जुळणारा टॅग
४५ ४५ This error usually means that for some reason, an opened tag in the XML did not have the same name as the tag that was opened, or a nested tag (inside the tag) was not closed correctly. या त्रुटीचा सहसा अर्थ असा होतो की काही कारणास्तव, एक्सएमएलमध्ये उघडलेल्या टॅगचे टॅग उघडलेले टॅगसारखे नाव नव्हते किंवा नेस्टेड टॅग (टॅगच्या आतील) योग्यरित्या बंद केलेला नाही.

५४ ५४ Other errors opening project प्रकल्प उघडताना इतर त्रुटी
५५ ५५ There are a number of other errors that you might see relating to the syntax of the AUP file not being quite correct, such as: अशा अनेक त्रुटी आहेत ज्या आपण कदाचित AUP धारिकाच्या सिंटॅक्सशी संबंधित पाहू शकता, अगदी बरोबर नाही, जसे की:
५६ ५६ unclosed token at line x (for example, the final </project> tag is missing its ending ">" लाइन एक्स वर अनलॉक केलेले टोकन (उदाहरणार्थ, अंतिम </ प्रकल्प> टॅगचा शेवट ">" गहाळ आहे
५७ ५७ no element found at line x (for example, the final </project> tag is missing in an otherwise correct file) रेष x वर कोणताही घटक आढळला नाही (उदाहरणार्थ, अंतिम </ प्रकल्प> टॅग अन्यथा योग्य धारिकामध्ये गहाळ आहे)
५८ ५८ junk after document element at line x x वर कागदजत्र घटकानंतर जंक
५९ ५९ The best advice in these cases is to examine the line quoted carefully, and don't manually edit the AUP file unless it is unavoidable. या प्रकरणांमधील सर्वोत्कृष्ट सल्ला म्हणजे कोट केलेली ओळ काळजीपूर्वक तपासणे आणि तो अपरिहार्य असल्याशिवाय AUP धारिका व्यक्तिचलितरित्या संपादित करू नका.
