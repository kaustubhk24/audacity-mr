4 4 This error message usually means that when reopening the AUP project file normally, Audacity finds an unexpected non-English, accented or control character at the line quoted in the error message. There are three main causes of this. या एरर मेसेजचा अर्थ असा होतो की एओपी प्रोजेक्ट धारिका सामान्यपणे पुन्हा सुरू करतांना ऑडिटीला एरर मेसेजमध्ये उद्धृत केलेल्या ओळीवर अनपेक्षित इंग्रजी, उच्चारण किंवा नियंत्रण वर्ण आढळले. याची तीन मुख्य कारणे आहेत.
५ ५ The project contains imported files whose metadata contains symbols that Audacity cannot display. This may create either the "(invalid token)" or "invalid character number" error प्रोजेक्टमध्ये आयात केलेल्या धारिका आहेत ज्यांच्या मेटाडेटामध्ये ओड्यासिटी प्रदर्शित होऊ शकत नाही अशी चिन्हे आहेत. हे एकतर "(अवैध टोकन)" किंवा "अवैध वर्ण संख्या" त्रुटी तयार करू शकते
६ ६ The project was saved containing non-English or accented characters using an old ANSI build of Audacity for Windows. ANSI builds are not properly Unicode aware, so do not support non-English or accented characters. विंडोजसाठी ओड्यासिटीच्या जुन्या एएनएसआय बिल्डचा वापर करून हा प्रकल्प इंग्रजी-नसलेल्या किंवा उच्चारणित वर्णांसह जतन झाला. एएनएसआय बिल्ड्स युनिकोड योग्यरित्या जागरूक नाहीत, म्हणून इंग्रजी-नसलेल्या किंवा उच्चारण केलेल्या वर्णांना समर्थन देऊ नका.
७ ७ The latest version of Audacity will always fully support Unicode. You can check your Audacity version at Help > About Audacity... (or Audacity > About Audacity on Mac). ओड्यासिटीची नवीनतम आवृत्ती युनिकोडला पूर्णपणे समर्थन देईल. आपण मदत> ओड्यासिटी बद्दल ... (किंवा ओड्यासिटी> मॅकवरील ओड्यासिटीबद्दल) वर आपली ऑडसीटी आवृत्ती तपासू शकता.
८ ८ Solution: उपाय:
९ ९ Make a copy of the AUP project file and open it in a text editor. You can use the default Notepad on Windows or TextEdit on Mac, but in a few instances these could change the AUP file in such a way that Audacity could no longer open it. एओपी प्रोजेक्ट धारिकाची एक प्रत बनवा आणि मजकूर संपादकात ती उघडा. आपण विंडोजवर पूर्वनिर्धारित नोटपॅड किंवा मॅकवरील टेक्स्ट एडिट वापरू शकता परंतु काही घटनांमध्ये एओपी धारिका अशा प्रकारे बदलू शकते की ओड्यासिटी यापुढे ती उघडू शकत नाही.
10 10 We recommend instead Notepad++ for Windows and Brackets or BBEdit for Mac. आम्ही त्याऐवजी विंडोज आणि कंस किंवा नोटसाठी बीएपीएडिटसाठी नोटपॅड ++ करण्याची शिफारस करतो.
11 11 Turn word wrap off in the menus (or in TextEdit, drag the window horizontally so that lines do not wrap). मेनूमध्ये शब्द लपेटणे बंद करा (किंवा TextEdit मध्ये विंडो आडव्या ड्रॅग करा जेणेकरून रेषा लपेटू नयेत).
12 12 In the line indicated in the error message, look for one of these: त्रुटी संदेशात दर्शविलेल्या ओळीत, यापैकी एक शोधा:
13 13 projname= प्रोजेमनाव =
14 14 name= नाव =
1५ 1५ value= मूल्य =
1६ 1६ title= then remove all characters inside the quotation marks that follow which are not a number or an A to Z letter. You could also change those characters to a number or A to Z letter. For example, a string of control characters you would want to remove or change might start with &# followed by four numbers and a शीर्षक = त्यानंतर अवतरण चिन्हातील सर्व अक्षरे काढा जी नंतर एक संख्या किंवा ए टू झेड अक्षरे नसतील. आपण ती वर्ण संख्या किंवा ए ते झेड पत्रात देखील बदलू शकता. उदाहरणार्थ, आपण हटवू किंवा बदलू इच्छित नियंत्रण वर्णांची स्ट्रिंग कदाचित & # सह प्रारंभ होईल आणि त्यानंतर चार संख्या आणि ए
1७ 1७ semi-colon, such as &# 0 0 1 3 ;. For symbols there will usually be five characters between the &# and a colon. अर्ध-कोलन, जसे & # 0 0 1 3;. चिन्हांसाठी सामान्यत: & # आणि विरामचिन्हे दरम्यान पाच वर्ण असू शकतात.
1८ 1८ Save the changes to the .aup file. बदल .aup धारिकामध्ये जतन करा.
1९ 1९ If you re-open the project and an error occurs at another line, repeat the steps above in the new line indicated in the error message. आपण प्रकल्प पुन्हा उघडल्यास आणि दुसर्‍या ओळीवर त्रुटी उद्भवल्यास, त्रुटी संदेशामध्ये दर्शविलेल्या नवीन ओळीत वरील चरण पुन्हा करा.
20 20 Here is a real-world example of a corrupt line in the GENRE tag in a project where "Country" and "Western" were incorrectly separated using the control character for a horizontal tab, also missing the closing semi-colon: एका प्रोजेक्टमधील जेएनआरई टॅगमधील भ्रष्ट रेषाचे वास्तविक जगातील उदाहरण येथे आहे ज्यात क्षैतिज टॅबसाठी नियंत्रण वर्ण वापरुन "देश" आणि "वेस्टर्न" चुकीचे विभक्त केले गेले होते, तसेच बंद होणारी अर्धविराम गहाळ आहे:
21 21 <tag name="GENRE" value="Country &# 0 0 0 ९ Western"/> <टॅग नाव = "GENRE" मूल्य = "देश आणि # 0 0 0 ९ पाश्चात्य" />
22 22 And the same line corrected so that the two words can be read into the project: आणि तीच ओळ दुरुस्त केली जेणेकरुन दोन शब्द प्रकल्पात वाचता येतील:
23 23 <tag name="GENRE" value="Country, Western"/> <टॅग नाव = "GENRE" मूल्य = "देश, पाश्चात्य" />
24 24 If you had to rename "projname" in the AUP file, rename both the AUP project file and the corresponding _data folder so that they have the same name as "projname" in the AUP file. आपल्याला एओपी धारिकामध्ये "प्रोजेमनेम" पुनर्नामित करायचे असल्यास, एयूपी प्रोजेक्ट धारिका आणि संबंधित _डेटा फोल्डर या दोहोंचे नाव बदला जेणेकरून ते एपीपी धारिकामध्ये "प्रोजेमनाम" सारखेच नाव ठेवू शकतील.
