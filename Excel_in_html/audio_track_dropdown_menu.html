६0 ६0 Format स्वरूप
६१ ६१ Converts the Sample Format of the current track state to १६-bit integer PCM, २४-bit integer PCM or ३२-bit floating point. The number of computer bits representing each sample in the waveform will be recalculated for the changed precision. Further edits will be performed at the new bit depth, but audio data for any track states before the conversion remains at its previous bit depth. This means that undoing the format change is always lossless. वर्तमान गीतपट्टा स्थितीचे नमुना स्वरूप १६-बिट पूर्णांक पीसीएम, २४-बिट पूर्णांक पीसीएम किंवा ३२-बिट फ्लोटिंग पॉईंटमध्ये रूपांतरित करते. बदललेल्या सुस्पष्टतेसाठी तरंगरूप मधील प्रत्येक नमुन्याचे प्रतिनिधित्व करणारे संगणक बिटची गणना केली जाईल. पुढील संपादने नवीन बिट खोलीवर केली जातील, परंतु कोणत्याही गीतपट्टासाठी ध्वनी डेटा रूपांतरण आधीच्या थोडा खोलीत राहील. याचा अर्थ असा की स्वरुपाचा बदल पूर्ववत करणे नेहमीच हानी नसते.
६२ ६२ Rate दर
६३ ६३ Change the Sample Rate of the track, which can be thought of as the concentration of audio samples in the track or the time distance between them. Increasing the rate squeezes the existing samples into a shorter length (so increasing playback speed and pitch), while decreasing the rate spreads them out over a greater length (so reducing playback speed and pitch). This is useful for example if you import a file that has the wrong rate and thus plays at the wrong speed. गीतपट्टाचा नमुना दर बदला, ज्याचा विचार गीतपट्टामधील ध्वनी नमुन्यांची एकाग्रता किंवा त्या दरम्यानची वेळ अंतर म्हणून केला जाऊ शकतो. दर कमी केल्याने विद्यमान नमुने कमी लांबीमध्ये पिळले जातात (त्यामुळे प्लेबॅकचा वेग आणि खेळपट्टी वाढत आहे), तर दर कमी केल्याने त्यांचे प्रसार मोठ्या लांबीवर होईल (त्यामुळे प्लेबॅकचा वेग आणि खेळपट्टी कमी होईल). उदाहरणार्थ आपण चुकीची दर असलेली धारिका आयात केली आणि अशा प्रकारे चुकीच्या वेगाने प्ले केल्यास हे उपयुक्त आहे.
