९ ९ Audio Host ध्वनी होस्ट
१० १० Selects the particular interface with which Audacity communicates with your chosen playback and recording devices. विशिष्ट मुखपृष्ठ निवडतो ज्याद्वारे ओड्यासिटी आपल्या निवडलेल्या प्लेबॅक आणि ध्वनिमुद्रण उपकरणसह संप्रेषण करते.
११ ११ On Windows, the choice is between the following audio interfaces: विंडोजवर, निवड खालील ध्वनी मुखपृष्ठ दरम्यान आहे :
१२ १२ MME: This is the Audacity default and the most compatible with all audio devices. एमएमईः हे ओड्यासिटी पूर्वनिर्धारित आहे आणि सर्व ध्वनी उपकरणसह सर्वात अनुकूल आहे.
१3 १3 Windows DirectSound: This is more recent than MME with potentially less latency. विंडोज डायरेक्टसाऊंडः संभाव्यत कमी विलंब असलेल्या एमएमईपेक्षा हे अगदी अलीकडील आहे.
१4 १4 Windows WASAPI: This host is the most recent Windows interface between applications (such as Audacity) and the audio interface driver. WASAPI was first officially released in २००७ in Windows Vista. WASAPI is particularly useful for "loopback" recording of computer playback. २4-bit recording devices are supported using this host. Playback is usually emulated. As a result, the playback slider in Mixer Toolbar will only scale the system playback slider's current level up or down rather than directly manipulating that system slider. विंडोज WASAPI : हे होस्ट अनुप्रयोग (जसे की ओड्यासिटी) आणि ध्वनी मुखपृष्ठ ड्राइव्हर दरम्यान सर्वात अलीकडील विंडोज मुखपृष्ठ आहे. WASAPI प्रथम अधिकृतपणे २००७ मध्ये विंडोज व्हिस्टामध्ये प्रसिद्ध झाले. खासकरुन संगणक प्लेबॅक ध्वनिमुद्रणसाठी "लूपबॅक" करण्यासाठी वासा उपयुक्त आहे. हे होस्ट वापरुन २4-बिट ध्वनिमुद्रण उपकरण समर्थित आहेत. प्लेबॅक सहसा अनुकरण केले जाते. याचा परिणाम म्हणून, मिक्सर साधनपट्टीमधील प्लेबॅक स्लाइडर थेट सिस्टम स्लाइडरमध्ये फेरफार करण्याऐवजी सिस्टम प्लेबॅक स्लाइडरची सद्य पातळी खाली किंवा खाली स्केल करेल.
