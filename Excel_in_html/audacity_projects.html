४ ४ All editing in Audacity is done as a project. It is important to note that a project is not a single audio file but a complex set of files managed by Audacity. ओड्यासिटी मधील सर्व संपादन प्रकल्प म्हणून केले जाते. तो एक प्रकल्प म्हणजे एकच ध्वनी धारिका नसून ओड्यासिटी द्वारे व्यवस्थापित धारीकांचा जटिल संच आहे हे लक्षात घेणे महत्वाचे आहे.
५ ५ Audacity projects have a top-level AUP file and a related folder containing the sound files (the audio data). The AUP is not a sound file, it is merely a list of instructions that tell Audacity how to construct the project from the sound files in the folder. Audacity projects can only be used by Audacity. ओड्यासिटी प्रकल्प एक उच्च स्तरीय एयुपी(AUP) धारिका आणि ध्वनी धारिका (ध्वनी माहिती) असलेली संबंधित निर्देशिका आहे. एयुपी(AUP) एक ​​आवाज धारिका ते फक्त कसे निर्देशिका ध्वनी धारिका प्रकल्प बांधकाम ओड्यासिटी सांगतो की सूचना सूची आहे नाही आहे. ओड्यासिटी प्रकल्प फक्त ओड्यासिटी मध्येच वापरली जाऊ शकते.

Projects are a convenient way to save your extended multi-track piece and come back to it later exactly as you left off. सर्व ओड्यासिटी गीतपट्टा आणि फीत, नाव, मोठेपणा लिफाफा गुण वाढणे आणि पॅन माहिती, इतर प्रकल्प माहिती एकत्र जतन करणे. ध्वनी मुद्रण आयात धारिका, व्युत्पन्न ध्वनी किंवा त्या कोणत्याही यांचे मिश्रण असू शकते. प्रकल्प आपल्या विस्तारित बहु-गीतपट्टा तुकडा जतन करण्यासाठी आणि नंतर आपण सोडले त्याच म्हणून परत एक सोयिस्कर मार्ग आहेत.
२२ २२ Audacity's project format is not compatible with and cannot be opened in any other audio application. ओड्यासिटी प्रकल्प स्वरूप सुसंगत नाही आणि कोणतेही अन्य ध्वनी अनुप्रयोग मध्ये उघडले जाऊ शकत नाही.
२३ २३ The structure of a saved Audacity project is: एक जतन ओड्यासिटी अशी या प्रकल्पाची रचना आहे:
२४ २४ An AUP project file - the name of the project followed by ".AUP", for example एक एयुपी(AUP) प्रकल्प धारिका - प्रकल्प नाव ".एयुपी(AUP)" त्यानंतर, उदाहरणार्थ
२५ २५ "my_song.AUP" "My_song.एयुपी(AUP)"
२६ २६ A _data folder with the same project name and in the same folder as the .AUP file, for example "my_song_data" त्याच प्रकल्प नाव आणि .एयुपी(AUP) धारिका प्रमाणे निर्देशिका मध्ये एक _data निर्देशिका, उदाहरणार्थ "my_song_data" साठी
२७ २७ Within that _data folder, a sub-folder structure with lots of little AU files which are individual segments of the audio data. की _data निर्देशिका, ध्वनी माहिती वैयक्तिक विभागाच्या आहेत थोडे AU धारिका बरेच उप-निर्देशिका संरचना आत आहेत.
२८ २८ Note that default behavior on Windows and Mac is to hide certain file extensions ; if this applies, the .AUP file will only appear as "my_song" in the computer. On Windows, the .AUP extension will be hidden if Audacity was installed from the .exe installer and "Hide file extensions for known file types" remains checked in Windows Explorer. You can unhide extensions on Windows by following these instructions. लक्षात ठेवा विंडोज व Mac वर मुलभूत वर्तन काही निश्चित धारिका विस्तार लपविण्यासाठी आहे; या लागू होत असल्यास, .एयुपी(AUP) धारिका फक्त संगणक मध्ये "my_song" म्हणून दिसेल. ओड्यासिटी .exe इंस्टॉलर आणि "ज्ञात धारिका प्रकार करीता लपवा धारिका विस्तार" राहते विंडोज एक्सप्लोरर मध्ये चेक वरून स्थापित केले होते तर विंडोज वर, .एयुपी(AUP) विस्तार लपविला जाईल. आपण या सूचनांचे अनुसरण करून विंडोज विस्तार समक्ष आणू शकतात.
२९ २९ The project file describes how Audacity links these AU files together to make up the clips and tracks in the project; it also contains gain, pan and envelope information, data to manage the प्रकल्प धारिका ओड्यासिटी प्रकल्प फिती आणि गीतपट्टा अप करण्यासाठी एकत्र या AU धारिका दुवे कसे वर्णन; तो देखील वाढणे, पॅन आणि लिफाफा माहिती समाविष्टीत आहे, व्यवस्थापित करण्यासाठी माहिती
३० ३० waveform display and carries links to any dependent audio files being read directly from their source location. तरंग प्रदर्शन आणि कोणत्याही अवलंबून ध्वनी धारिका दुवे असतो त्यांच्या स्रोत स्थान थेट वाचली आहे.
३१ ३१ The AUP file is in XML format and can be opened in a text editor if required. The AU files are stored in a lossless, uncompressed format. Their default size is १ MB or less. This Audacity Project Format is designed to make editing audio faster in Audacity. एयुपी(AUP) धारिका XML स्वरूपात आहे आणि आवश्यक असल्यास एक टेक्स्ट एडिटर मध्ये उघडली जाऊ शकतात. ऑस्ट्रेलिया धारिका एक निराश, संकुचित स्वरूपात साठवले जातात. त्यांचे पूर्वनियोजित आकार १ एमबी किंवा कमी आहे. या ओड्यासिटी प्रकल्प स्वरूप ओड्यासिटी जलद ध्वनी संपादन करण्यासाठी योजलेले आहे.
३२ ३२ By updating individual AU files during editing, Audacity can change audio or move it around in the project without copying large quantities of data from one place to another. संपादन दरम्यान वैयक्तिक AU धारिका अद्यतनित करून, ओड्यासिटी ध्वनी बदलू किंवा एक ठिकाणाहून दुसर्या माहिती मोठ्या प्रमाणात नक्कल न करता प्रकल्प सुमारे हलवू शकता.

३६ ३६ Never rename the AUP file or the _data folder by hand. एयुपी(AUP) धारिका किंवा हाताने _data निर्देशिका पुनर्नामित करू नका.
३७ ३७ If you want to rename your project (for example, to save a snapshot at a particular point), use one of the File > Save Project > Save Lossless Copy of Project... or File > Save Project > Save Project As... commands. आपण आपल्या योजनेचे नाव बदलणे करू इच्छित असल्यास, धारिका> एक प्रकल्प जतन करा> प्रकल्प जतन करा निराश प्रत करा ... किंवा धारिका> प्रकल्प जतन करा > जतन करा प्रकल्प म्हणून वापर (उदाहरणार्थ, एका विशिष्ट टप्प्यावर एक स्नॅपशॉट जतन करण्यासाठी) ... आदेश.

३९ ३९ Saving an Audacity Project एक ओड्यासिटी प्रकल्प जतन करीत आहे
४० ४० A saved Audacity Project can only be opened and used by Audacity. If you want an audio file that will play on your music player or to burn a CD, use an Export command. जतन ओड्यासिटी प्रकल्प फक्त उघडले आणि ओड्यासिटी वापरली जाऊ शकते. आपण आपल्या संगीत उपकरणे चालवणार किंवा CD बर्ण करण्यासाठी एक ध्वनी धारिका इच्छित असल्यास, निर्यात आदेशचा वापर करा.
४१ ४१ The benefits of saving an Audacity Project are: एक ओड्यासिटी प्रकल्प बचत फायदे आहेत:
४२ ४२ Saving a project lets you save unfinished work and re-open it later in Audacity exactly as it was, with all edits and recorded/imported tracks preserved. Note carefully that the Undo history is not saved with the project and so the project history starts afresh when you re-open the project later. एक प्रकल्प जतन करीत आहे अपूर्ण काम आणि पुन्हा-उघडा तो नक्की म्हणून नंतर ओड्यासिटी मध्ये जतन सर्व संपादने आणि मुद्रण / आयात गीतपट्टा जतन करतो देते. टीप पूर्ववत इतिहास प्रकल्प आणि प्रकल्प आपण नंतर प्रकल्प पुन्हा उघडता तेव्हा इतिहास सुरू पुन्हा म्हणून जतन केला नाही काळजीपूर्वक आहे.
४३ ४३ Audio data is always preserved in lossless quality. This is useful if you have already exported to a lossy audio format like MP३ but decide to edit the project further. Editing and re-exporting the project saves the additional quality loss of re-editing the previously exported MP३. ध्वनी माहिती नेहमी निराश गुणवत्ता संरक्षित केला जातो. आपण आधीच MP३ सारख्या हानिकारक ध्वनी स्वरूप निर्यात पण पुढील प्रकल्प संपादित करण्यासाठी निर्णय असेल तर हे उपयुक्त आहे. संपादन आणि पुन्हा निर्यात प्रकल्प मागे निर्यात MP३ पुन्हा संपादित अतिरिक्त गुणवत्ता नुकसान वाचवतो.
४४ ४४ No need to re-import or re-record files पुन्हा आयात किंवा पुन्हा मुद्रण धारिका करण्याची गरज नाही
४५ ४५ Fast loading, even of multiple long tracks जलद लोड, अगदी अनेक लांब गीतपट्टा

If the project is not saved, the necessary audio data is stored in the temporary folder specified in the Directories section of Preferences until exiting the application. At that point, Audacity offers the choice of saving a project or not. एक प्रकल्प जतन करण्याची गरज नाही तोपर्यंत तो तात्पुरत्या प्रकल्प धारिका काम करणे शक्य आहे आणि नंतर फक्त धारिका> निर्यात> निर्यात ध्वनी ... आवश्यक ध्वनी धारिका निर्यात करण्यासाठी वापरता आवश्यक आहे. प्रकल्प जतन न केल्यास, आवश्यक ध्वनी माहिती अनुप्रयोग विद्यमान पर्यंत प्राधान्ये पुस्तिका विभागात निर्दिष्ट तात्पुरती निर्देशिका मध्ये साठवली जाते. त्या ठिकाणी, ओड्यासिटी एक प्रकल्प जतन निवड देते किंवा नाही.

Audio data is only copied in for any parts of the file that you subsequently edit. Do not delete "-old" file(s) until you are happy with the exported file and no longer require the Audacity project. मूळ स्थान संकुचित धारिका वाचा (जलद): ओड्यासिटी वापर मागणीनुसार लोड करीत आहे. या आयात धारिका बरेच जलद केवळ आयात धारिका संदर्भ आणि आयात धारिका उपलब्ध उर्वरित अवलंबून असलेल्या लहान auf "सारांश" धारिका लिहून ठेवतात. ध्वनी माहिती केवळ धारिका आपण त्या त्यानंतर संपादन कोणत्याही भागात मध्ये प्रत केले आहे. आपण आयात धारिका संपूर्ण ध्वनी संपादन लागू केल्यास, ओड्यासिटी त्या वेळी संपूर्ण ध्वनी माहिती प्रत होईल, त्यामुळे आपण संपूर्ण ध्वनी की संपादित करा पूर्ववत करा करणार नाही असे प्रदान, प्रकल्प आयात धारिका अवलंबून यापुढे सदैव होईल आहे .लक्षात ठेवा आपण एकाच धारिका नाव आणि स्थान निर्यात करून वाचन-थेट डब्ल्यूएवी(WAV) किंवा एआयएफएफ धारिका खोडून पुन्हा तर, ओड्यासिटी एक प्रत्यय असलेली "-old" आणि एक नंबर आयात धारिका पुनर्नामित करते, नंतर या "-old" धारिका प्रकल्पासाठी अवलंबून धारिका होते. आपण निर्यात धारिका आनंदी आहेत आणि यापुढे ओड्यासिटी प्रकल्प आवश्यक पर्यंत "-old" धारिका (चे) हटवू नका.
९० ९० Do not choose the "Faster" option unless you completely understand the implications. If you move, rename or delete the imported file, the project data may be lost. आपण पूर्णपणे अभिप्रेत अर्थ समजून नाही तोपर्यंत "जलद" पर्याय निवडू नका. आपण हलविल्यास, पुनर्नामित करा किंवा आयात धारिका हटवा, प्रकल्प माहिती गमावला जाऊ शकतो.

१११ १११ Autosave and Recovery स्वयंचलितरित्या जतन आणि पुनर्प्राप्ती
११२ ११२ As soon as you open a project window and change the window contents, whether this is a previously saved project or you are creating a new project, Audacity writes a temporary reference file for that project called an ".autosave" file. This allows Audacity to automatically recover unsaved changes in the event of a crash or power loss. तितक्या लवकर आपण एक प्रकल्प विंडो उघडा आणि हा एक पूर्वी जतन प्रकल्प आहे किंवा आपण एक नवीन प्रकल्प तयार आहेत का, विंडो सामग्री बदलू म्हणून, ओड्यासिटी प्रकल्प एक ".autosave" धारिका म्हणतात, एका अस्थायी संदर्भ धारिका लिहितात. या ओड्यासिटी आपोआप बंद किंवा शक्ती नुकसान झाल्यास जतन न केलेले बदल पुनर्प्राप्त करण्यासाठी परवानगी देते.
११३ ११३ When you use Save Project, Save Lossless Copy of Project... or Save Project As... there are no longer unsaved changes, so the ".autosave" file is deleted until fresh unsaved changes are made. The file is stored in the "AutoSave" folder inside Audacity's folder for application data. आपण प्रकल्प जतन करा, प्रकल्प जतन करा दोषरहित प्रत करा ... किंवा जतन प्रकल्प म्हणून वापर करता, तेव्हा ... यापुढे जतन न केलेले बदल, ताज्या जतन न केलेले बदल केले जातात होईपर्यंत म्हणून ".autosave" धारिका हटविली आहे. धारिका अनुप्रयोग माहिती ओड्यासिटी च्या निर्देशिका आत "स्वयंजतन(AUTOSAVE)" निर्देशिका मध्ये साठवली जाते.
११४ ११४ The file is of identical internal structure to an AUP project file, but is not readable by a text editor. धारिका एयुपी(AUP) प्रकल्प धारिका एकसारखे अंतर्गत रचना आहे, पण एक टेक्स्ट एडिटर वाचनीय नाही.
११५ ११५ Most changes you make to the audio data, such as adding or removing audio or applying an effect to it, trigger writing the ".autosave" file. A change to the position of the selection or editing cursor does not write to the ".autosave" file, and is not captured when closing the project, unless the project is already "dirty" (as shown by Save Project in the submenu being not grayed out and being clickable). अशा धारिका ".autosave" लेखन ट्रिगर जोडून किंवा ध्वनी काढून टाकणे किंवा त्यावर परिणाम अर्ज, म्हणून आपण ध्वनी माहितीमध्ये सर्वाधिक बदल. किंवा निवड स्थानावर बदल कर्सर संपादित ".autosave" धारिका लिहू शकत नाही, आणि प्रकल्प बंद करताना पकडले नाही, प्रकल्प सबमेनूसह नाही असल्याने जतन प्रकल्प करून दर्शविल्याप्रमाणे "गलिच्छ" (आधीच आहे तोपर्यंत राखाडी आणि क्लिक जात).
११६ ११६ Disk space usage डिस्क स्पेस वापर
११७ ११७ This can be particularly important if you are editing a large project. आपण एका मोठ्या प्रकल्प संपादित करीत आहेत तर हे महत्वाचे असू शकते.
११८ ११८ Audacity references the original audio material until you actually perform some kind of edit on it, such as cutting a piece away or using any effect on it. So if you Edit > Duplicate a selection, this doesn't actually increase the number of .au files in the _data folder, and doesn't increase disk space usage. Edit > Copy for example does increase the number of .au files, but this is so these files can be retained separately on the Audacity clipboard for Edit > Paste. ओड्यासिटी संदर्भ मूळ ध्वनी सामग्री आपण प्रत्यक्षात संपादन काही प्रकारचे अशा दूर एक तुकडा कापून किंवा कोणताही परिणाम वापरून म्हणून काम होईपर्यंत. त्यामुळे आपण संपादित करा> निवड डुप्लिकेट तर, या प्रत्यक्षात संख्या .au _data निर्देशिका धारिका वाढत नाही, आणि डिस्क स्पेस वापर वाढत नाही. संपादित करा> उदाहरणार्थ प्रत .au धारिकांची संख्या वाढ नाही, पण त्यामुळे या धारिका संपादित ओड्यासिटी फीतबोर्ड वर स्वतंत्रपणे कायम राखणे शक्य आहे> पेस्ट करा.
११९ ११९ Once you perform an edit, the original unchanged audio is retained, along with the changed audio. This is so you can use Edit > Undo and Edit > Redo to try out edits and undo them if they do not sound as you wanted. आपण संपादन सुरू एकदा, मूळ जसाच्या तसा ध्वनी बदललेल्या ध्वनी सोबत ठेवली जाते. त्यामुळे आपण संपादने बाहेर प्रयत्न संपादित करा> पूर्ववत करा आणि संपादित करा> पुन्हा वापरता आणि ते पूर्ववत करू शकत आपल्याला पाहिजे ते आवाज नाही तर हे आहे.
१२० १२० If you edit just a small selection in a track, only that small selection of changed data is written to disk. Nonetheless, this still represents a doubling of disk space usage for that selection, and a further edit on that selection (unless you first undo the previous edit) adds the same amount again, and so on. आपण एक गीतपट्टा मध्ये फक्त एक लहान निवड फक्त त्या बदलेल्या डाटा लहान निवड डिस्कवर लिहिले आहे की, संपादन, तर. तथापि, हे अद्याप निवड डिस्क जागा वापर दुप्पट, आणि त्या निवड वर आणखी संपादन प्रतिनिधित्व (आपण प्रथम मागील संपादित करा पूर्ववत करा नाही तर) पुन्हा पुन्हा तीच रक्कम जोडते, आणि तसेच चालू राहते.
१२१ १२१ For example, a five minute stereo recording at default quality settings takes १०० MB of space. If you edit the whole length of that recording, disk space usage will increase to २०० MB. उदाहरणार्थ, मुलभूत गुणवत्ता समायोजन येथे पाच मिनिट स्टिरीओ मुद्रण १०० जागा एमबी(MB) घेते. आपण त्या मुद्रण संपूर्ण लांबी संपादित करत असल्यास, डिस्क स्पेस वापर २०० एमबी(MB) पर्यंत वाढवण्यात येईल.
१२२ १२२ If disk space is a problem while working on a project, you can use View > History... to discard Undo levels and reclaim disk space. Note that once you close the project by using File > Close or exiting Audacity, the audio data needed for Undo/Redo is discarded, so as to make project storage as economical as possible. एका प्रकल्पावर काम करत असताना डिस्क जागा एक समस्या आहे, तर, आपण पूर्ववत करा पातळी आणि पुन्हा हक्क डिस्क जागा टाकून पहा> इतिहास वापरू शकता .... लक्षात ठेवा की आपण> धारिका वापरून बंद करा किंवा ओड्यासिटी विद्यमान प्रकल्प बंद एकदा, पूर्ववत / पुन्हा आवश्यक ध्वनी माहिती त्यामुळे शक्य आर्थिकदृष्ट्या प्रकल्प स्टोरेज म्हणून, काढून टाकली जाते.
१२३ १२३ Any audio data placed on the Audacity clipboard by Cut or Copy is however retained until you exit Audacity itself. आपण ओड्यासिटी स्वतः बाहेर पडू पर्यंत कट किंवा प्रत करून ओड्यासिटी फीतबोर्ड स्थीत कोणतीही ध्वनी माहिती मात्र ठेवली जाते.
